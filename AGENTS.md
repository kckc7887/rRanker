# AGENTS.md

每次执行变更必须：

1. 读取本文件最新版本
2. 执行变更
3. 在 `changelog/` 下新建 `YYYY-MM-DD_主题.md`，包含：改动原因、具体实现、期望输出、实际输出
4. 提交时在 commit message 末附 Agent 标识

产品想法与待办管理：

1. 用户提出功能构想、产品规划、暂缓实现项时，优先记录到顶层 `TODO.md`
2. 记录时保留用户原始意图，拆成：目标、用户价值、实现思路、依赖/风险、当前优先级、暂缓原因
3. 不要把构想直接实现成代码，除非用户明确要求开始实现
4. 如果 `TODO.md` 不存在，先创建再记录

不要：假定依赖存在、猜测 API 行为、跳过验证。
