# M1 设计需求清单

> 2026-07-12 产出。本清单是 M1 双端查分 MVP 开工的硬门禁之一（见 `docs/development-readiness.md` 第 2.C 节）。用户逐项确认或修改后，M1 才进入实现阶段。
>
> 每项给出"默认建议"（基于 M0 现状与 ROADMAP 技术栈）和"待用户确认点"。用户只需对每项回答"同意"或给出修改意见即可。

## 1. 视觉方向

**默认建议**：M1 继续沿用 M0 功能线框风格（#F7F8FA 背景 + #246BFD 主色 + 白色卡片），不做最终品牌视觉；最终视觉延后到 M4 发布准备阶段。

**待用户确认**：

- 是否同意 M1 保持线框风格？
- 若不同意，用户是否能提供 Figma / 设计稿 / 参考应用截图？

## 2. 配色系统

**默认建议**（沿用 M0）：

| 用途 | 色值 |
| --- | --- |
| 页面背景 | `#F7F8FA` |
| 卡片背景 | `#FFFFFF` |
| 主色（按钮/Tab 激活/Rating 数字） | `#246BFD` |
| 主文本 | `#111827` |
| 次文本 | `#6B7280` |
| 弱文本/占位 | `#9CA3AF` |
| 边框/分割 | `#D1D5DB` |
| 危险/清除 | `#B42318` |
| B35 标签 | 主色 `#246BFD` |
| B15 标签 | 待定（建议用主色派生色，如 `#7C3AED`） |

**待用户确认**：

- 上述色值是否保留？
- B35 / B15 是否需要用不同颜色区分标签？（M0 当前都用主色）
- 是否需要难度色（BAS/ADV/EXP/MAS/Re:MAS 各一色）？建议参考舞萌官方难度色。

## 3. 字体与字号

**默认建议**：使用系统字体（iOS SF / Android Roboto），不引入自定义字体；日文长标题用 `numberOfLines={1}` + ellipsis。

| 用途 | 字号 | 字重 |
| --- | --- | --- |
| 页面大标题（玩家名） | 28 | 700 |
| 卡片标题 | 18 | 700 |
| 列表标题 | 16 | 600 |
| Rating 数字（总览） | 42 | 800 |
| Rating 数字（列表项） | 18 | 800 |
| 正文 | 14 | 400 |
| 辅助文本 | 12 | 400 |

**待用户确认**：

- 字号阶梯是否保留？
- 是否需要引入第三方字体（如 maimai 风格字体）？引入会增加 bundle 体积。

## 4. 间距系统

**默认建议**：使用 4 的倍数间距阶梯。

| 用途 | 值 |
| --- | --- |
| 页面边距 | 16 |
| 卡片内边距 | 18 |
| 卡片间距 | 14 |
| 列表项间距 | 10 |
| 组件内元素间距 | 8 / 12 |

**待用户确认**：间距阶梯是否保留？

## 5. 组件规范

**默认建议**：M1 抽以下共用组件：

- `Card`：白底、圆角 14、内边距 18。
- `PrimaryButton` / `SecondaryButton` / `DangerTextButton`：圆角 10、内边距 13。
- `TextInput`：边框 1、圆角 10、内边距 12。
- `RatingBadge`：B35/B15 标签 + Rating 数字。
- `QueryStateView`：统一处理 loading / error / empty / stale / data 五态。
- `TabBar`：沿用 Expo Router 默认底栏 + Ionicons。

**待用户确认**：

- 组件清单是否够用？
- 卡片圆角是否统一为 14？M0 当前有 12/14/16 三种，需统一。

## 6. 页面布局（5 个 tab）

### 6.1 总览（index）

- 顶部：玩家名 + 数据来源标签。
- 主卡片：DX Rating（5 位补零）+ B35/B15 数量。
- 数据状态卡：来源、更新时间、stale 提示。
- M1 新增：刷新按钮（触发 TanStack Query refetch）。

### 6.2 B50

- 列表头部：说明文本。
- 列表项：序号 + 标题 + 分组标签(B35/B15) + 类型(SD/DX) + 等级 + Rating。
- M1 新增：按 Rating 排序的固定顺序；可考虑分组分割线（B35 / B15）。

### 6.3 成绩

- 列表头部：总数。
- 列表项：标题 + 类型 + 等级 + 版本 + 达成率 + Rating。
- M1 新增：筛选栏（难度 / 版本 / 类型 / 排序方式），筛选状态进 Zustand。

### 6.4 查歌（新增）

- 顶部：搜索输入框（按曲名 / songId）。
- 列表项：封面 + 标题 + artist + 版本 + 难度列表。
- M1 范围：仅曲名 + songId 搜索；复合筛选（曲师/难度/定数/版本）延后到 M2。
- 数据来源：`getSongs()`（fixture M1 补全 + 水鱼 `/music_data`）。

### 6.5 设置

- 沿用 M0：水鱼账密登录 / Import-Token / 清除本机数据。
- M1 新增：当前会话状态显示（已登录/未登录/会话过期）、数据来源切换说明、Import-Token 获取路径文案。

**待用户确认**：

- 5 个页面的布局是否同意？
- 成绩页筛选栏放在列表顶部还是抽屉？
- 查歌页是否需要封面图？（需要确认封面 CDN 许可与缓存策略）

## 7. 状态展示

**默认建议**：统一 `QueryStateView` 处理 5 态：

| 状态 | 视觉 |
| --- | --- |
| loading | 居中 ActivityIndicator |
| error | 居中文案 + 重试按钮 |
| empty | 居中文案（如"暂无成绩"） |
| stale | 顶部黄色横幅"数据可能过期，下拉刷新" + 旧数据 |
| offline | 顶部灰色横幅"离线模式，显示最近缓存" + 旧数据 |

**待用户确认**：

- 5 态视觉是否同意？
- stale 与 offline 是否合并为一种提示？

## 8. 列表性能

**默认建议**：

- 成绩页用 `FlatList`（M0 已用），600+ 成绩时开启 `getItemLayout`、`removeClippedSubviews`、`initialNumToRender=15`。
- B50 页用 `FlatList`（50 项，无需特殊优化）。
- 查歌页用 `FlatList` + 本地筛选，曲库 500+ 条时同上优化。

**待用户确认**：性能参数是否需要更激进？

## 9. 平台差异

**默认建议**：

- 安全区：用 `react-native-safe-area-context`（M0 已引入）。
- 手势返回：iOS 用 `react-native-screens` 的 `fullScreenGestureEnabled`；Android 用系统返回键。
- 系统字体：不覆盖，用默认。
- 状态栏：`expo-status-bar` `dark` 模式（M0 已用）。

**待用户确认**：是否有其他平台差异需要处理？

## 10. 暗色模式

**默认建议**：M1 不做暗色模式，延后到 M4。

**待用户确认**：是否同意延后？

## 11. 图标与封面

**默认建议**：

- 图标：`@expo/vector-icons` Ionicons（M0 已用）。
- 封面：M1 暂不显示封面图，查歌页只用文字列表；封面 CDN 许可与缓存策略确认后再加。

**待用户确认**：

- 是否同意 M1 不显示封面？
- 若要显示，封面 CDN 来源是水鱼还是华立？许可需确认。

## 12. 动效

**默认建议**：M1 不引入显式动效，列表滚动用默认；下拉刷新用 Expo 默认。

**待用户确认**：是否需要刷新/切换 tab 的过渡动效？
