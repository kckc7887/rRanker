# Win11 + iPhone 17 测试方案

> 当前测试主 App 的查询、计算、缓存、展示，以及好友码到本地/水鱼/落雪的写入编排。真实外部账号写入只在用户已有授权的测试账号上人工执行；自动测试不修改外部账号。

## 1. 双端共用边界

可以共用并在两端验证：

- React Native 页面和组件。
- TypeScript 领域模型、Rating/B50、筛选和推荐算法。
- provider/repository/application service 接口。
- TanStack Query 请求状态、SQLite 数据模型和安全存储调用方式。

这里的 service 是 App 进程内 TypeScript 模块，不是 localhost HTTP 服务。当前不启动后台服务器，也不测试代理。

## 3. 每次改动的最低测试

在 Windows 执行：

1. TypeScript 类型检查。
2. Rating、B35/B15、排序、筛选、ID 映射单元测试。
3. Provider 契约测试：正常、缺字段、未知枚举、超时、限流、4xx/5xx、损坏 JSON。
4. 组件状态：加载、空数据、完成、离线缓存、错误恢复。
5. Android Emulator 冒烟：启动、导航、返回键、键盘和长列表。

在 iPhone 17 + Expo Go 执行：

1. 安全区、手势返回、键盘和系统字体。
2. 水鱼真实读取、断网、网络切换和 token 失效。
3. 切后台、锁屏、系统回收后重新打开。
4. SQLite 缓存、离线查看和 schema 迁移。
5. 日文长标题、封面失败和 600+ 条成绩滚动。

## 4. 功能阶段验收

### M0 领域底座

- fixture 完全脱敏。
- 同一输入在所有测试环境产生完全相同的 Rating/B50。
- 未知字段不会导致整批成绩失败。

### M1 双端查分 MVP

- iPhone Expo Go 与 Android Emulator 展示相同页面和数值。
- 水鱼 provider 与 fixture provider 可以无感切换。
- 不存在 localhost 服务或代理；上传取数依赖公网 score-hub Friend VS。

### M2 查询与工具箱

- 查歌、筛选、Rating、容错和牌子计算都有边界测试。
- 数据源失败时仍可查看最近有效快照。
- 所有页面明确显示数据来源和最后更新时间。

#### M2A 曲库查询

- 用曲名、数字 ID、别名、曲师和谱师分别搜索，并组合 SD/DX、难度、定数、歌曲版本和谱面版本筛选。
- 检查日文长标题、无别名、无成绩、封面失败占位图以及 1300+ 曲库滚动。
- 从列表进入歌曲详情，核对谱面定数、谱师、物量和本地最佳成绩。

#### M2B 工具箱

- Rating 固定定数核对全档结果，并反推可达/不可达目标至 0.0001%。
- 容错检查零 BREAK、零物量、混合判定和 BREAK 细分；确认页面注明“用于估算”。
- 牌子检查空难度、多 requirement、rate/FC/FS 边界和缺失歌曲跳转。
- 版本名称对照检查 `舞萌DX 2026 ↔ maimai でらっくす PRiSM PLUS`；逐版本总结检查所有版本均有卡片，且无成绩时平均值显示“无成绩”。

#### M2C 稳定性与双端边界

- 分别断开水鱼和 LXNS，确认成绩、详细曲库、别名、姓名框按资源独立回退且来源状态正确。
- 使 token 返回 401/403，确认保留最近快照并提示重新登录，不切换 fixture。
- 检查筛选即时响应、搜索短防抖、600+ 成绩和 1300+ 曲库滚动。
- iPhone 检查安全区、键盘、返回手势、后台恢复和网络切换；Android 检查返回键、键盘、后台恢复和网络切换。

### M3 收藏、分享与推荐

- 收藏、练习清单和偏好在重启后保留。
- 导出结果不泄露 token 或隐藏字段。
- 推荐结果可解释，输入相同时结果稳定。

#### M3A 收藏、练习清单、本地标签与备份

- 在查歌页收藏歌曲，从总览进入“我的曲库”，检查收藏数量、列表和取消收藏同步更新。
- 在歌曲详情把具体谱面加入练习清单，并分别为歌曲、谱面添加、去重和删除标签；检查标签过滤。
- 重启 App、退出水鱼登录并切换 fixture，确认个人数据仍保留且不随账号切换。
- 断开曲库网络或导入包含当前曲库不存在歌曲的备份，确认显示歌曲 ID 和“个人数据已保留”。
- 导出 JSON，确认不包含 token、cookie、玩家账号、成绩记录或曲库缓存；取消系统分享后可继续操作。
- 在 iPhone 与 Android 之间分别导入同一备份，检查默认合并的状态/标签并集和显式替换结果。
- 导入损坏 JSON、未知版本、额外字段与超过 1 MiB 的文件，确认原有个人数据不变。
- 点击“清除本机凭据和缓存”，分别验证“仅凭据与缓存”保留个人数据，“同时删除个人数据”清空全部本机数据。
- iOS 27 beta 3 从总览进入“我的曲库”，确认左上角仅显示固定返回箭头，不出现 `(tabs)` 标签；再进入任意歌曲并往返，确认返回按钮不重复跳动。
- Android 16.0 / API 36.1 在查歌页反复收藏/取消同一歌曲，确认空心与实心爱心来自同一轮廓、位置和尺寸不变化。

### M4 双端稳定

- iPhone 完成长周期真实使用。
- Android Emulator 与云设备完成回归。
- 没有 Android 真机时明确标记“Android 未完成真机验收”。
- 没有 Apple Developer 账号时明确标记“iOS 未完成分发包验收”。

## 5. 本地/测试查分器与水鱼/LXNS 读写测试

### 本地与测试查分器

- 冷启动默认存在本地查分器、满分测试查分器和原空数据测试游戏；三者都应显示在设置页账号管理中，切换后重启可恢复上次活跃的内置账号。
- 在设置页连续添加两个本地玩家并分别改名；为两者同步不同好友码后，反复切换并重启，确认名称、成绩、B50、Rating 和曲库来源按账号独立保存，不互相覆盖。
- 默认本地玩家首次为空且不可删除；额外本地玩家可删除，删除其中一个不影响默认玩家和其他本地玩家。
- 本地玩家好友码同步后的成绩、B50、Rating 与曲库快照均可在断网重启后读取。
- 本地总览只显示“同步数据”，按钮和下拉刷新均打开原上传弹窗；弹窗全量展示目标但本次仅勾选本地，关闭后普通页面恢复原持久勾选。
- 测试查分器覆盖全部未禁用 SD/DX 谱面（包含 locked），验证 101%、SSS+、AP+、FDX+；有物量时 DXScore=总物量×3，无物量谱面仍保留且显示空 DXScore；已有详细曲库缓存时断网切换到测试玩家仍可显示成绩。

### 水鱼

- 每个实际使用端点分别保存 request/response schema 与错误样例。
- 验证认证方式、token 失效、限流和不可查询账号。
- B50、全量成绩、曲库和谱面统计不能假定使用相同权限。
- 上游数据异常不得覆盖最近有效快照。

### 落雪

- 无需凭据的 LXNS 公共曲库已用于歌曲、谱面和版本元数据，需覆盖正常、超时、损坏响应和缓存回退。
- OAuth 玩家读取与成绩上传已接入；自动测试覆盖 PKCE、读取、Bearer 请求体、刷新 token 轮换、401/403、429/5xx、取消和部分成功。
- 真实上传只使用已有授权账号人工验证：先记录目标账号原状态，上传一条可识别成绩，确认落雪与 App 内刷新结果后停止，不在自动测试或他人账号执行。
- LXNS 曲库失败时不得把无法确认版本的成绩静默归入 B35；优先使用最近有效曲库或完整成绩快照。

### 好友码多目标流程

- 一次取数同时勾选本地、水鱼与落雪，确认只创建一次好友/成绩任务，并按列表顺序写入目标。
- 模拟中间目标失败，确认已成功目标不回滚，后续目标继续；结果逐项显示成功、失败、跳过数量和“写入成功但应用内刷新失败”。
- 在等待好友、获取成绩、目标写入与上传后刷新阶段分别取消，确认不再开始新目标，已完成目标保留。
- JWT 水鱼、失效落雪 OAuth 和测试查分器不可勾选；本地始终可写。

## 7. Expo Go 的结论边界

- Expo Go 可以验证当前选定的 JS/UI 和受支持 Expo SDK 能力。
- Expo Go 验证通过不等于 TestFlight/App Store 包验证通过。
- 若后续某个非上传功能确实需要自定义原生模块，单独评估是否调整实现或开通 Apple Developer 账号。
- 在此之前，不为未来爬虫预装原生依赖。

## 8. 发布前仍需补做

- iOS：Apple Developer 账号、development/preview build、TestFlight 安装与升级。
- Android：至少一台真实设备、Play 内部测试、厂商后台与电池策略。
- 发布验证不阻塞当前功能开发，但必须在宣称对应平台正式可用前完成。
