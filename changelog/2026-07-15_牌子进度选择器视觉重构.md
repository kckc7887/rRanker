# 牌子进度选择器视觉重构

## 改动原因

版本牌子选择列表嵌套长列表观感差、信息密度低；需在保留「单框展开 / 选完收起 / 仅版本牌子」交互的前提下重做版式。

## 具体实现

1. 选择器改为单一卡片：收起时突出当前牌子名；展开后版本用网格 chip，档位用横排 chip，选定后仍整框收回。
2. 进度区改为百分比 + 进度条 + 完成数；缺失曲目优先显示曲库曲名，并附歌曲 ID。
3. 展开时默认定位到当前已选版本前缀。

## 期望输出

页面以一框选择 + 进度卡 + 缺失列表为主，不再出现臃肿的多级菜单行；交互不变。

## 实际输出

牌子进度页 UI 已按上述版式重构，交互与分组逻辑保持不变。表头改用带 `gap` 的容器，避免收起后选择卡与进度卡贴合。
