# M3A 双端导航与收藏图标修复

## 改动原因

用户在 iPhone 17 / iOS 27 beta 3 与 Android 16.0 / API 36.1 首轮验证 M3A 时发现：iOS 从“我的曲库”进入歌曲以及返回时，左上角返回按钮会重复跳动，“我的曲库”的返回标签还暴露了内部路由名 `(tabs)`；Android 使用 Unicode `♡/♥` 表示收藏状态，不同系统字体下两者轮廓不一致。

## 具体实现

- 根 Stack 统一设置 `headerBackButtonDisplayMode: 'minimal'`，使 iOS 返回按钮固定为仅箭头，不再根据前一页标题计算宽度。
- 关闭 iOS 返回按钮历史菜单，避免 beta 系统为返回标题准备额外布局。
- 在根 Stack 静态声明 `library/index` 与 `songs/[songId]` 的标题，移除两个页面内部的动态 `Stack.Screen` 更新。
- 将查歌收藏图标从 Unicode 字符替换为 Ionicons 同一图标族的 `heart-outline/heart`，点击区域固定为 44×44。
- Jest 中对矢量图标做最小 mock，保留收藏交互和可访问性标签测试。
- 把用户实际系统版本和两项回归步骤写入 M3A 双端测试清单。

## 期望输出

- iOS “我的曲库”左上角不再显示 `(tabs)`，只显示稳定的返回箭头。
- iOS 在“我的曲库”与歌曲详情之间往返时，返回按钮不发生重复跳动。
- Android 收藏/取消收藏只改变爱心填充状态，轮廓、尺寸和位置保持一致。

## 实际输出

- `npm test` 通过：Vitest 23 个文件 78 项，Jest Expo 5 个套件 18 项。
- `npm run typecheck` 与 `npm run lint` 通过，无错误或警告。
- Android Hermes 静态导出成功：1579 个模块，主包 5.01 MB；临时导出目录已清理。
- 根 Stack 路由配置和 Android 图标资源均完成静态与打包验证；iOS 27 beta 3 的原生动画表现仍需用户在同一设备复验后确认。
