# 2026-07-13 M1 收口与 LXNS 曲库接入

## 改动原因

M1 双端运行验证后复核发现，真实水鱼成绩仍使用 fixture 的 `M0 Current Version` 计算 B35/B15，导致当前版本成绩可能全部误入 B35。与此同时，会话只在进入设置页后恢复、离线缓存状态未正确显示、Import-Token 未实际联网验证、一次刷新重复读取玩家与成绩，且 QueryStateView 五态测试没有执行。

## 具体实现

- 将数据源拆分为水鱼成绩 provider 与 LXNS 公共曲库 provider。
- 使用 LXNS 数值型版本表和谱面级版本，通过规范化歌曲 ID、类型、难度序号匹配 B35/B15；无法匹配的成绩明确排除并显示数量。
- 新增 LXNS 曲库 SQLite 缓存，升级成绩快照 schema，旧 v1 快照安全失效。
- 根布局在首屏前恢复 SecureStore 会话，失败时回落 fixture 并显示错误。
- 账密和 Import-Token 均在成功读取水鱼玩家与成绩后才保存和激活。
- 去除 provider 内重复 B50 请求，查询缓存键不再包含凭据原文。
- 保留 Vitest，并按 Expo 官方路径加入 jest-expo，启用 QueryStateView 五态组件测试。
- 更新 API 协议、开发状态、移动端 README 和 TODO 数据源边界。

## 期望输出

- 真实成绩按 LXNS 谱面版本准确进入 B35/B15。
- 每次刷新只读取一次水鱼玩家、一次水鱼成绩和一次 LXNS 曲库。
- LXNS 失败时使用曲库缓存；水鱼失败时展示最近完整快照。
- 无效 Token 不落盘，持久会话冷启动不闪现 fixture。
- 单元测试、UI 测试、类型检查与 lint 全部通过。
