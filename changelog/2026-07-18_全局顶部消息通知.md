# 全局顶部消息通知

## 改动原因

原有成功、失败、校验和危险操作确认均直接调用 React Native `Alert.alert`，Android 与 iOS 的视觉不一致，也无法统一控制动画、排队和消息时长。用户要求把当前及后续所有原生弹窗统一为从顶部向下出现的消息通知样式。

## 具体实现

- 新增应用级 `NotificationProvider` 与 `useNotification`，支持成功、信息、警告、错误四种通知，以及取消、普通、危险三类操作按钮。
- 在根布局挂载通知宿主，通知适配顶部安全区和窄屏/平板宽度，使用顶部滑入/滑出动画、状态图标、圆角、阴影和手动关闭按钮。
- 普通消息按类型使用 3 秒或 5 秒默认时长；关键提醒和确认操作保持显示，操作通知使用轻遮罩阻断背景并支持 Android 返回取消。
- 使用 FIFO 队列保证一次只显示一条；动作执行与关闭均有防重复保护。
- 将工具置顶、成绩图导出、账号管理、总览同步和上传页中的 14 处 `Alert.alert` 全部迁移到全局通知。
- 新增 ESLint 限制，禁止后续从 `react-native` 导入 `Alert`。
- 更新设置、上传、总览、工具箱和成绩图测试，并新增通知组件的时长、排队、安全区、持久提示、背景阻断及返回键测试。

## 期望输出

- 当前业务不再出现系统原生 Alert，普通消息从顶部滑入并按规则消失。
- 连续消息依次展示，危险操作保留明确确认和取消路径。
- 后续直接使用 React Native Alert 时 lint 立即失败，引导开发者使用统一通知 API。
