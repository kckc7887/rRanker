# Phigros 拉取最新云存档

## 改动原因

用户对照发现 App 同步到的是**去年旧存档**（RKS 约 15.x），而游戏内云同步后已是 **16.17**。问题不在解析算法，而是**取错了云存档记录**或**下载了 CDN 缓存的旧 zip**：

1. `getGameSave` 固定使用 `results[0]`，LeanCloud 多条 `_GameSave` 时可能不是最新
2. UI 将 `updatedAt` 写成当前时间，无法看出存档实际日期
3. 下载 `gameFile.url` 未穿透 CDN/本地 HTTP 缓存

## 具体实现

1. **`getGameSave`**：`order=-updatedAt&limit=20`，`pickLatestGameSave` 取最新有效条目；请求 `cache: no-store`
2. **`downloadSave`**：附加 `?_ts=<updatedAt>` 并禁用缓存
3. **`PhigrosScoreProvider`**：缓存 `saveMeta`；`getSaveUpdatedAt()` 供 UI 展示真实存档时间
4. **`use-game-data`**：成绩来源时间改为云存档 `updatedAt`

## 期望输出

- 同步后成绩/RKS 与游戏内云存档一致（16.17）
- 总览/成绩页「更新时间」显示 LeanCloud 存档时间，而非误导性的“刚刚”

## 实际输出

- `phigros-auth.test.ts` 3 项 + 既有单测通过
