# 总览上传页：好友码 → score-hub → 水鱼

## 改动原因

按 M5 口径把总览「上传数据」从占位接成可用流程：好友码取第一方成绩，勾选已绑定查分器后写入托管服务。

## 具体实现

- `UploadDataSheet`：好友码、可勾选查分器列表、开始上传、状态区；`wait_acceptance` 时 Alert。
- SecureStore 持久化好友码与勾选（`upload-prefs-store`）。
- `score-hub-client` / `upload-maimai-from-friend-code`：登录 → 慢轮询 → update_score → latest sync。
- `score-hub-sync-map` + `diving-fish-upload`：曲库拼 title 后 Import-Token 上传水鱼。
- 仅水鱼 + import-token 可勾选；落雪/本地预览/JWT 禁用。

## 期望输出

绑定 Import-Token 的水鱼账号后，总览可上传；同意 Bot 好友后状态推进并写出成绩到水鱼。

## 实际输出

代码与单测已落地。依赖公网 score-hub，单次任务慢轮询以降低负担；未接落雪上传，不把 hub sync 写入主查分缓存。
