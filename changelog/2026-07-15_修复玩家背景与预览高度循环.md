# 修复玩家背景与预览高度循环

## 改动原因

成绩图片预览存在两个运行问题：落雪账号可能继续复用升级前不含 `frameId` 的成绩缓存，导致无法显示玩家当前背景；自定义背景使用负边距扩展模糊区域，但 HTML 自动高度把背景装饰层也当作内容测量，画布高度和背景高度相互推动，造成预览高度持续增长。

## 具体实现

- 为实际参与排版的玩家资料块增加 `data-layout-content` 标记，自动高度、`ResizeObserver` 和动态节点监听只处理这类内容。
- 背景与色调层继续随画布铺满，但不再参与内容高度计算；未来成绩行需要自动撑高时同样使用排版内容标记。
- 升级游戏数据 React Query 查询键版本，使应用更新后不复用旧的内存查询结果。
- 将 SQLite 成绩快照结构版本从 2 升级到 3，自动删除缺少玩家背景元数据的旧账号快照并重新向落雪读取。
- 增加 HTML 测量边界和旧快照失效的回归测试。

## 期望输出

- 落雪账号重新加载后能从玩家资料的 `frame.id` 显示当前背景。
- 选择任意自定义背景后，预览画布保持最小 3:4，并只随真实前景内容增高，不再无限增长。
- 旧成绩缓存能平滑失效，不影响新快照的正常保存和读取。
