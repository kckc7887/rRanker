# 原生底部导航评估

## 改动原因

用户希望确认现有 5 项底部导航能否像页面返回与多页面导航一样，改为 Android/iOS 各自的原生组件。本次先完成技术可行性确认和产品待办记录，避免在未获得明确实施指令前改变主导航。

## 具体实现

- 核对移动端当前使用 Expo Router JavaScript Tabs。
- 核对已安装的 Expo Router 6.0.24 包含 SDK 54 的 `expo-router/unstable-native-tabs`，无需新增底栏依赖。
- 查阅 Expo SDK 54 Native Tabs 文档，记录 alpha 稳定性、静态 Tab、Android 选中图标和跨平台视觉差异等约束。
- 在 `TODO.md` 中记录迁移目标、用户价值、实施思路、依赖风险、优先级和暂缓原因。

## 期望输出

形成可直接执行的原生底栏迁移边界，同时保持现有 5 个入口与当前运行代码不变，等待用户明确授权实施。

## 实际输出

已确认方案可行：可用当前依赖将底栏迁移到 Expo Router Native Tabs，并继续由根级原生 Stack 管理二级页面。本轮仅更新产品待办和变更日志，未修改应用代码。
