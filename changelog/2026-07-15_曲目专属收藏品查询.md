# 2026-07-15 曲目专属收藏品查询

## 改动原因

歌曲详情需要展示「只有该曲可作为条件」的收藏品；落雪虽无按曲 ID 反查接口，但四类收藏品列表带 `required[].songs`，可本地过滤。版本牌子等多曲条件会自然排除。

## 具体实现

- 领域：`CollectionItem` / `CollectionSnapshot`，`collectionsForSong`（曲目 ID 并集恰好等于当前曲）。
- Provider：`LxnsCatalogProvider.getCollections()` 并行拉取 `trophy|icon|plate|frame` 的 `/list?required=true`；fixture / empty 补齐。
- Hook：`useCollections` + SQLite 资源键 `collections`；账号切换时随 `invalidateAccountDataQueries` 失效。
- UI：歌曲详情「歌曲信息」上方「收藏品」卡片；头像/姓名框/背景用 CDN 预览，称号显示色标与文案。
- 文档：`docs/api-protocol.md` 回填四类列表与 2026-07-15 验证记录。

## 期望输出

- 打开有曲目专属头像/称号的歌曲详情时，卡片列出对应收藏品且不含版本「极/将/神/舞舞」。
- 无匹配时显示「无曲目专属收藏品」；加载失败可单独重试且不阻塞曲库。
- 单元测试覆盖过滤与 provider 合并。

## 实际输出

- 上述代码与单测已落地；真实 LXNS 探测确认 `?required=true` 与单曲 `required.songs` 形态可用。
