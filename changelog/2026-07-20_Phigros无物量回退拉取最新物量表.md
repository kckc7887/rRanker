# Phigros 无物量时回退拉取最新物量表

## 改动原因

曲库若未带上物量（`current.noteCounts` 缺失、首拉失败或缓存空表），详情页会直接显示「物量未提供」，未再尝试拉取对象存储上的最新物量表。

## 具体实现

1. `PhigrosCatalogProvider.loadNoteCounts`：依次尝试 `current.noteCounts`、版本约定路径 `phigros/releases/{ver}/metadata/note_counts.tsv`；仍无数据时再拉一次最新 `current.json` 并重试。
2. `usePhigrosCatalog` 在每次 query 前 `resetCatalogCache()`，使 React Query refetch 真正重新请求 OSS。
3. 歌曲详情：整首歌均无物量时自动 refetch；拉取中显示「加载物量中…」，仍无则「物量未提供」。

## 期望输出

- 无物量时先尝试最新物量表，成功则展示 TAP/HOLD/DRAG/FLICK/总计。
- 确实不存在时才显示「物量未提供」。

## 实际输出

- 相关单测与 UI 测试通过；changelog 与中文 commit 已落地。
