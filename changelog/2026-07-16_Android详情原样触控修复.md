# Android 详情页原样触控修复

## 改动原因

此前为规避 Android 歌曲详情页顶部悬浮区域拦截触控，临时改用了非透明原生标题栏，导致顶部封面不再铺满、页面外观偏离原设计；同时详情页中使用 `react-native-gesture-handler` 的按钮在 Android 上仍无法响应，包括长文本展开、版本切换、谱面类型切换、练习清单、搜索谱面确认以及标签增删。

## 具体实现

- 恢复透明导航栏与沉浸式顶部封面，保持原有颜色、尺寸、间距和按钮视觉样式。
- 将返回与收藏按钮拆为两个独立的 40×40 绝对定位触控区域，不再使用横跨屏幕顶部的高层容器，避免其拦截下方页面触控。
- Android 详情页的普通操作按钮改用 React Native 原生 `Pressable`，对应容器改用普通 `View`；iOS 继续沿用现有手势组件实现。
- 保留谱面卡片横向轮播所需的手势根容器与手势滚动组件，不改变轮播布局和交互方式。
- 标签编辑器在 Android 上同样使用原生 `Pressable`，恢复删除标签与添加标签操作。
- 补充 Android 回归测试，覆盖返回、收藏、长文本展开、版本名称切换、DX/SD 谱面切换、练习清单、搜索谱面确认、删除标签和添加标签。

## 期望输出

- Android 歌曲详情页维持原有沉浸式样式，歌曲封面铺满顶部。
- 左上返回和右上收藏可点击，且不会阻挡页面其他区域。
- 详情页所有操作按钮均可正常点击；iOS 现有正常行为不受影响。
- Android 构建能够正常完成。

## 实际输出

- 已恢复透明导航栏、顶部满幅封面和原有按钮视觉样式。
- 已将 Android 详情页及标签编辑器中的普通按钮切换为原生触控实现，并移除会覆盖整行的顶部触控容器。
- 类型检查和 ESLint 均通过。
- Vitest 40 个测试文件、172 项测试全部通过。
- Jest UI 回归 7 个测试套件、31 项测试全部通过。
- Android Expo 导出成功，Metro 完成 1798 个模块打包。
