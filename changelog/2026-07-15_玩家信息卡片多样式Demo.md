# 玩家信息卡片多样式 Demo

## 改动原因

现有玩家信息卡片仍然偏大，且视觉方向尚未确定。用户要求卡片只占输出图片宽度的一半，并希望先通过多个 HTML 示例选择风格，再实现应用内样式与收藏品选择器。

## 具体实现

- 新增独立 `demo/player-card-styles/index.html`，不修改移动端正式预览实现。
- 提供机台原生、赛博 HUD、收藏票券、黑金铭牌四种视觉方案。
- 每种卡片都包含头像、完整姓名框、DX Rating、玩家名和称号，并严格设置为 3:4 输出画布宽度的 50%。
- 增加点击放大和单选候选交互，便于比较细节。
- 从落雪公共接口读取并复核示例收藏品，下载头像 `53603`、姓名框 `200503`，称号使用 `4299` 的名称与金色属性。
- 在 TODO 中记录正式实现所需的样式选择、头像/姓名框/称号全量选择器，以及列表规模、限流、缓存和懒加载风险。

## 期望输出

用户可直接打开一个 HTML 页面，在真实 50% 宽度比例下比较四种玩家信息卡片，选定或组合下一轮正式实现方向。

## 实际输出

HTML Demo、说明文档和本地落雪示例素材已生成。桌面浏览器实测四张卡片均按画布 50% 宽度渲染，头像、完整姓名框、Rating、玩家名和称号可见；候选单选与点击放大交互正常；浏览器控制台无错误或警告。正式 App 代码本轮未修改。
