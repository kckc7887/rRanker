# iOS 缓存标签页后台动画暂停

## 改动原因

- 标签页状态缓存没有生成多份页面快照，但会永久保留每个已访问页面的当前实例。
- 成绩卡片、Rating 标签和状态标签内存在多组使用原生驱动的 `Animated.loop`；此前它们只在组件卸载时停止，而缓存页面失焦后不再卸载，导致后台标签仍持续运行无限动画。
- 列表限窗减少了同时存在的动画数量，因此卡顿出现得更晚，但没有消除后台持续工作。

## 具体实现

- 为标签页缓存容器增加前台状态 Context；首次聚焦后仍只激活页面一次，失焦时仅发布后台状态，不卸载页面。
- 缓存首次激活时的页面根 React 元素，标签焦点切换不再向重页面根传递新元素，只有订阅前台状态的动画节点重新渲染。
- `ScoreVisuals` 和 `LayeredGradientBadge` 的原生无限动画增加前台条件：页面失焦时执行现有清理函数并停止动画，重新聚焦时从零创建一轮新动画。
- 默认 Context 为前台，使歌曲详情、图片生成等不在缓存标签页内的既有动画行为保持不变。
- 新增动画生命周期测试，覆盖首次聚焦启动、失焦全部停止、再次聚焦只恢复一轮、再次失焦继续停止；缓存页测试同时验证页面状态保留和重页面根不随焦点切换重渲染。

## 期望输出

- 任意时刻只有当前活动标签页的流光标签持续运行动画。
- 缓存页面失焦后不再占用持续的原生动画与合成资源，多次切换不会叠加动画循环。
- 回切页面仍保持原内容、滚动位置和组件内部状态，不重新出现卸载白闪。

- `npm run test:ui` 通过：16 个 UI 测试文件共 68 项全部通过。
- 新增的动画生命周期和缓存根渲染测试通过。
- `npm run typecheck` 通过。
- `npm run lint` 无错误，保留一条与本次无关的既有 import 顺序警告。
