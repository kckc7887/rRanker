# 修复 DX 封面资源 ID

## 改动原因

封面串行缓存进度能够正常完成，但预览仍有大量封面缺失。真实 CDN 请求证明问题不在请求并发，而在资源地址：成绩记录中的 DX 歌曲 ID 包含查分器偏移，例如 `11447`；落雪曲绘资源实际使用去除偏移后的 `1447`。直接请求 `11447.png` 返回 404，请求 `1447.png` 返回 200。

## 具体实现

- 封面 URL 生成接入领域层既有 `mapCoverId`，统一处理 `+10000`、`+100000`、`+110000` 等曲绘 ID 偏移。
- 卡片仍展示成绩源提供的原始歌曲 ID，仅封面 CDN 地址和缓存键使用规范化资源 ID。
- SD 原始 ID 与 DX 偏移 ID 指向同一资源时复用同一个内存 Promise 和 Expo Image 磁盘缓存，不重复请求或转换 Base64。
- 补充 `11447 → 1447`、`110123 → 123` 以及 SD/DX 共用缓存的回归测试。

## 期望输出

- 串行缓存进度完成后，DX 成绩卡片能够显示与对应 SD 歌曲相同的正确封面。
- 不再把查分器显示 ID 直接用于曲绘 CDN，也不改变卡片展示给用户的歌曲 ID。

## 实际输出

- 真实 CDN 验证：`11447.png` 为 404，`1447.png` 为 200、`image/png`。
- `npm run typecheck` 通过。
- 全量 Vitest：36 个测试文件、137 项测试全部通过。
- 全量 Jest UI：7 个测试套件、27 项测试全部通过。
- Android Expo 静态导出成功，修正后的曲绘寻址进入正式 bundle。
- ESLint 无错误；总览页保留 1 条既有 `useCallback` 依赖警告，与本次改动无关。
