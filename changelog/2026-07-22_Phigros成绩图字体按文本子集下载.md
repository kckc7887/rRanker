# Phigros 成绩图字体按文本子集下载

## 改动原因

根因排查确认：进入成绩图页会按固定 12 字体清单全量准备，导出门禁也绑死 12/12；「按需」仅指相对安装包内置的首次下载，从未按成图文本筛选。用户要求只下载有用字体。

## 具体实现

- 新增可见文本收集与 Unicode 脚本启发式映射：始终准备核心字体（`phi`、`Aldrich-Regular`），仅在文本命中对应脚本时追加扩展字体（日文假名、阿拉伯文、Emoji、数学符号等）。
- `preparePhigrosFonts` 支持 `neededNames`，只下载并校验所需子集；进度 `total` 改为子集长度。
- 注入 HTML 前裁剪模板 CSS 的多余 `@font-face` 与 `body` 字体栈，避免引用未下载字体。
- 导出门禁改为「所需字体就绪」；纯中文/基本拉丁场景通常只需 2 个核心字体即可导出。
- 补充单元测试与预览页回归（含假名曲名触发 `NotoSansJP`）。

## 期望输出

典型中文成绩图不再下载日文/Emoji 等扩展字体；含假名、Emoji 等文本时只追加对应扩展字体；导出仍在所需字体就绪后可用，且成图不因缺失字体回退到系统字体。

## 实际输出

- 定向单测：`phigros-font-cache`、`phigros-font-coverage` 通过。
- UI 测试：`phigros-best-image-preview` 3 项通过，断言 `neededNames` 仅含核心 + `NotoSansJP`（假名样例），导出门禁在所需字体完成后放开。
