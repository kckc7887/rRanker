# Android 歌曲详情全页点击修复

## 改动原因

上一轮为了修复 Android 歌曲详情页顶部返回和收藏按钮，将页面内绝对定位标题栏移动到内容之后并增加了较高的 `elevation`。真机反馈表明该 Android 原生 ViewGroup 进入整页命中排序后，虽然只有顶部可见，却拦截了详情页其余触控，导致除输入框外的返回、收藏、版本切换、DX/SD 切换、练习清单、谱面确认和标签按钮全部无法点击。上一轮对 Android 触控层的判断不完整，需要撤销该覆盖方案。

## 具体实现

- Android 不再渲染页面内绝对定位的 `SongDetailChrome`，从节点层面移除会覆盖详情内容命中区的高层 ViewGroup。
- Android 改用 Expo Router / React Navigation 的非透明原生标题栏：恢复原生返回按钮，并在原生 `headerRight` 中提供收藏按钮。
- Android 原生标题栏使用深色背景和白色返回图标，内容从标题栏下方开始布局，不再让沉浸式头图与原生标题栏竞争点击区域。
- iOS 继续保留已在真机稳定运行的透明沉浸式标题栏和页面内按钮，不改变 iOS 手势树。
- 移除页面内标题栏的 Android `elevation`，防止未来条件渲染或平台判断变化时再次形成高层触控覆盖。
- 新增 Android 分支结构回归测试，确认不挂载页面内返回覆盖层、原生标题栏非透明且显示返回入口，并验证版本切换、DX/SD 切换和收藏回调仍可执行。

## 期望输出

- Android 歌曲详情页所有按钮恢复点击，包括顶部返回/收藏、版本切换、难度卡片按钮、练习清单、谱面确认与标签操作。
- Android 顶部采用稳定的原生深色标题栏；iOS 继续保持原有沉浸式样式和正常交互。
- 纵向滚动与难度卡片横向滑动不受影响。
