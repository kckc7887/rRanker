# 落雪收藏品条件分析

## 改动原因

需要从落雪公共 API 获取当前全部舞萌收藏品，并基于真实响应判断头像、姓名框、称号和背景的获取条件是否存在可复用共性，避免后续收藏品进度功能猜测上游规则。

## 具体实现

- 保存四类 `required=true` 公共接口的原始 JSON 快照，并记录来源、数量和 SHA-256。
- 遍历全部收藏品及条件对象，统计结构化覆盖率、字段组合、难度、成绩门槛、曲目数量和无结构化条件描述类别。
- 新增完整分析文档，记录可复用条件模型、上游字段不一致和进度追踪建议。
- 更新 LXNS 公共收藏品协议的本次验证日期、数量与条件约束。

## 期望输出

得到可复查的全量收藏品数据，并明确哪些获取条件能由成绩自动计算、哪些必须依赖其他玩家状态，同时识别会导致错误进度的上游数据异常。

## 实际输出

四类共 4771 件收藏品已完整解析：1730 件带结构化条件，3041 件仅有描述。全部结构化条件均可归约为“曲目集合 + 难度集合 + 至多一种成绩门槛”，但同步门槛存在 `fc` / `fs` 字段混用；无结构化条件则以角色觉醒与区域/活动获得为主，不能仅凭成绩快照自动计算。四份 JSON 均通过语法验证并记录哈希。
