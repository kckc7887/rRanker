# Phigros 完整存档成绩列表

## 改动原因

此前 `getRecords()` 只返回 B30 计算结果中的 Best27（27 条），成绩页无法展示云存档中的全部游玩记录，与 phiTool 完整 `gameRecord` 语义不符。

## 具体实现

1. **`src/domain/phigros.ts`**
   - 新增 `collectScoredEntries()`：展开全部已游玩谱面并附加定数/RKS
   - 新增 `phigrosEntryToScoreRecord()` / `gameRecordToScoreRecords()`：映射为通用 `ScoreRecord`
   - `computeB30()` 复用 `collectScoredEntries()`，B30 逻辑不变
   - 难度映射修正为 EZ/HD/IN/AT → basic/advanced/expert/master

2. **`src/providers/phigros-score-provider.ts`**
   - 存档 ZIP 只拉取一次，缓存完整 `gameRecord` + 定数表
   - `getRecords()` 返回全部成绩；`getB30()` / `getBestSections()` 专供 RKS 与最佳页

3. **`src/hooks/use-game-data.ts`**
   - `bestSections` 改为 Phi3 + Best27 两分区，不再从完整成绩里 slice(0,30)

4. **`app/(tabs)/records/index.tsx`**
   - 成绩查询不再额外拉 B30

5. **`tests/phigros-save.test.ts`**
   - 新增「全部谱面 vs 仅 B27」断言

## 期望输出

- 成绩页展示存档中全部已游玩谱面（数百条量级）
- 最佳/RKS 仍基于 Phi3 + Best27
- 测试通过
