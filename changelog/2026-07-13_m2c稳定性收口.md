# M2C 稳定性收口

## 改动原因

详细曲库、别名、姓名框和水鱼成绩来自不同链路，不能用 TanStack Query 的时间性 stale 代替离线状态，也不能因单项缓存损坏或认证过期清空其他有效数据。

## 具体实现

- 新增统一来源状态组件，成绩、B50、查歌、歌曲详情、牌子和版本页展示来源、更新时间及在线/缓存/不可用状态。
- 详细曲库、别名、姓名框使用独立 SQLite 资源键和 schema；损坏资源只删除自身。
- 水鱼 401/403 时继续保留最近完整成绩快照，并在来源标签提示重新登录，不切回 fixture。
- 详细曲库可用但别名无缓存时仍展示曲库并标记别名暂不可用；各资源有缓存时独立回退。
- 查歌和成绩列表使用 memo 化与 FlatList 虚拟化；搜索短防抖、筛选切换即时。
- React Native AppState 接入 TanStack Query focusManager，应用回到前台时恢复刷新；补充完整 M2 双端手验清单。

## 期望输出

外部链路分别失败时保留仍有效的数据；缓存状态可解释；认证过期不泄漏或覆盖凭据；600+ 成绩和 1300+ 曲库保持可用；双端边界有明确验收清单。
