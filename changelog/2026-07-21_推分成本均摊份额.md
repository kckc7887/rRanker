# 修正推分「期望成本」为份额均摊

## 改动原因

「期望成本 N」不是取前 N 条推荐，而是愿意用 N 首歌分摊精确加值：例如涨约 0.1、成本 4 首 → 每首承担约 0.025 总 RKS。

## 具体实现

1. `findPushRecommendations` 改为 `songCost`：`perSongShare = (exactTarget - currentRks) / songCost`，二分目标为 `currentRks + perSongShare`。
2. 列表返回全部可达谱面（按 Acc 差值升序），不再用 N 截断。
3. 推分页文案改为「期望成本（首歌）」，并展示精确加值 / 分摊 / 每首承担。

## 期望输出

- 加值 0.1、成本 4 → `perSongShare ≈ gainNeeded/4`（约 0.025）。
- 成本越大，单曲目标 Acc 越低。

## 实际输出

- `phigros-push.test.ts` 覆盖均摊与 Acc 降低；测试通过。
