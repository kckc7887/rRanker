# M4 单曲详情页沉浸标题与动态渐变修正

## 改动原因

用户查看单曲详情页视觉首版后指出四项理解偏差：曲绘应延伸到标题栏背后；封面黑色遮罩应为透明度渐变而非实色块；国服/日服名称点击没有在真机生效；所谓“流动”必须是动态移动的渐变而非静态彩虹。同时用户确认区域可以读取 LXNS 公共 API。

## 具体实现

- 为歌曲详情覆盖原生 Stack 配置，启用透明标题栏、白色最小返回按钮和浅色状态栏，使全宽正方形曲绘从屏幕顶端开始并覆盖标题栏区域。
- 使用 `expo-linear-gradient` 将曲绘底部文字遮罩改为顶部 100% 透明、底部 60% 透明，不再显示完整矩形黑块。
- 将版本切换按钮从横向文字 ScrollView 的触摸区域中拆出，解决真机滚动手势吞掉点击的问题。
- 新增 `localizedVersionName`：优先按 `versionId` 匹配，也可按当前国服/日服名称匹配旧缓存；`舞萌DX 2026` 与 `maimai でらっくす PRiSM PLUS` 可双向切换。
- 按 Expo SDK 54 官方兼容版本安装 `expo-linear-gradient ~15.0.8` 与 `@react-native-masked-view/masked-view 0.3.2`。
- 达成率动态彩虹改为 MaskedView 内部的双周期渐变带，以原生驱动持续横向位移；静态彩虹改为连续渐变，不再逐字换色。
- FC+、AP+、FS/FS+、FDX+ 标签改为循环横向流动的绿色、蓝色或金色渐变背景。
- 按 LXNS 官方契约把可空开放字符串 `Song.map` 映射为本地 `Song.region`；字段缺失、`null` 或空白时不显示区域列。
- 更新 API 契约、版本名称单元测试、LXNS provider 测试和详情页 UI 测试，覆盖旧缓存名称匹配、区域显示和动态渐变分支。

## 期望输出

- 首屏曲绘与系统标题栏形成连续沉浸式头图，返回按钮仍清晰可用。
- 黑色可读性区域自然渐隐，不再像一块矩形色块。
- 点击版本控件后实际切换国服/日服名称。
- 100.5% 及指定 FC/FS 状态能看到持续横向流动的渐变，而不是静态彩虹。
- 有 `map` 的歌曲展示区域，无区域数据时不占列。
