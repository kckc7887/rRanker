# Phigros 成绩图样式与页面对齐

## 改动原因

此前 Phigros 成绩图片采用独立设计，生成页面的控件顺序、尺寸和交互也没有与舞萌板块保持一致，导致导出结果与参考项目差异较大，两个游戏的生成体验存在割裂。

## 具体实现

- 按 phi-plugin 的 Apache-2.0 `resources/html/b19` 模板适配 Phigros 成绩图，使用 1200px 基准画布、斜切玩家资料条、右侧 Clear/FC/φ 统计、三列成绩卡、Phi 黄光和 Best 白光等原有视觉结构。
- 将模板接入 rRanker 的玩家资料、RKS、课题模式、同步时间、曲绘、谱面数据和分页模型，并继续支持 1080、1440、2160 三档宽度、素材内嵌、WebView 就绪协议和逐页 PNG 导出。
- 将 Phigros 生成页重构为与舞萌生成页相同的页面顺序和组件规格：分段类型选择、自定义 BestN 面板、样式列表、分辨率卡片、尺寸信息、预览框、分页点、导出按钮和 WebView 状态提示。
- 样式选择行增加头像和背景缩略图，素材选择弹窗统一使用胶囊选项和列表卡片交互。
- 新增第三方声明及 Apache-2.0 许可副本，明确模板来源、上游署名、rRanker 修改内容与 GPL 业务代码隔离边界。
- 补充 HTML 结构断言和 Phigros 生成页 UI 回归测试。

## 期望输出

- Phigros 导出图片直接呈现参考项目 B30 模板的布局和视觉语言，不再使用原先的独立圆角卡片方案。
- Phigros 与舞萌生成页在信息层级、控件位置、间距和预览导出交互上保持一致。
- 原有筛选、素材偏好、分页和三档分辨率导出能力继续正常工作。

## 实际输出

- 已完成参考模板结构适配与舞萌页面 UI 对齐，Best30 和自定义类型均复用新的三列成绩图模板。
- `npm run typecheck` 通过。
- `npm run lint` 通过，0 个错误；保留 6 个与本次修改无关的既有警告。
- `npm test` 通过：Vitest 66 个文件、326 个测试；Jest 21 个文件、95 个测试全部通过。
- 代表性 HTML 输出已通过 Phi 卡片、玩家资料、统计区、1200px 缩放与三档分辨率结构断言。
