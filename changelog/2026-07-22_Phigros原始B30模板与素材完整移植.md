# Phigros 原始 B30 模板与素材完整移植

## 改动原因

此前的 Phigros 成绩图只参考了 `astrbot_plugin_phi` 的信息结构和视觉语言，仍属于重新实现，无法保证与参考项目输出完全一致。参考项目由用户所有，因此本次按要求直接移植原始模板和素材，同时继续让成绩图片配置页沿用舞萌板块的布局与交互。

## 具体实现

- 将参考项目 `resources/html/b19`、`common`、`otherimg` 下的 62 个文件完整复制到移动端资源目录，包含原始 CSS、12 份字体、课题模式图标、评价图标、默认背景及相关图片；文件内容保持不变，仅将 `HIMALAYA.TTF` 的文件名后缀改为 Metro 可识别的小写 `.ttf`。
- 按参考模板 `b19.art` 的 DOM 层级和 class 名生成玩家信息、课题模式、统计表、Phi3 / Best27 成绩卡与页脚，直接注入原始 `snow.css`、`common.css`、`b19.css`，不再使用近似复刻样式。
- 评分等级、FC、课题模式和默认背景直接使用参考项目 PNG；玩家头像、歌曲曲绘和用户选择的动态背景仍由当前账号与 OSS 曲库数据注入。
- 新增 Expo Asset 模板资源加载器，将原始 CSS、字体和图片解析为 WebView 可读取的本地 URI，并为 Android / iOS WebView 配置本地文件访问范围。
- 新增 Metro 配置以识别 CSS 静态资源；Jest 增加 CSS 模块桩，保证原始模板资源进入生产包且不影响测试环境。
- 保留舞萌式 Phigros 生成页组件顺序、筛选、样式选择、分辨率、分页预览和逐页 PNG 导出流程。
- 更新第三方声明，记录 Apache-2.0 模板与素材的直接使用方式及唯一的文件名大小写调整。

## 期望输出

- Phigros B30 成绩图使用参考项目原有布局、字体、图标、装饰与样式，不再呈现 rRanker 自行近似实现的视觉。
- 三档分辨率、分页预览和 PNG 导出仍可正常使用；生成页操作方式继续与舞萌板块保持一致。
- 原始模板资源能够在 Expo SDK 54 的 Android / iOS 生产包和 WebView 中稳定加载。
