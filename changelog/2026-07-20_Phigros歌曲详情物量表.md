# Phigros 歌曲详情物量表

## 改动原因

对象存储已发布 `note_counts.tsv`（由 `current.json.noteCounts` 指向），但曲库与歌曲详情未消费，难度卡片无法展示 Tap/Hold/Drag/Flick 物量。

## 具体实现

1. 新增 `PhigrosChartNotes`，`Chart.notes` 支持舞萌与 Phigros 联合类型。
2. `loadNoteCountsTable` 解析 TSV，剥离 chart ID 的 `.0` 后缀后与 catalog 对齐。
3. `PhigrosCatalogProvider` 拉取 `noteCounts` 并写入各难度 `chart.notes`；拉取失败时曲库仍可用。
4. `PhigrosSongDetail` 难度卡片在谱师后展示只读物量表（样式对齐舞萌，不跳转容错工具）。

## 期望输出

- 有物量的难度卡显示 TAP / HOLD / DRAG / FLICK / 总计。
- 无物量时显示「物量未提供」。
- 单测覆盖 TSV 解析与详情页表格。

## 实际输出

- 单元测试与 UI 测试通过；changelog 与中文 commit 已落地。
