# Phigros 发布脚本上传谱面

## 改动原因

`demo/phigros-resource-publisher` 自用发布脚本此前参照 phiTool 默认配置，解包时关闭了 chart 提取，整理与上传流程也未包含谱面 JSON，对象存储中缺少 chart 资源。

## 具体实现

1. 新增 `phigros_publisher/extract_cli.py`：publisher 自有解包入口，配置与 [Phigros_Resource](https://github.com/7aGiven/Phigros_Resource) 一致（`Chart = true`，music 关闭），直接调用 `gameInformation.run` 与 `resource.run` 提取谱面。
2. `extractor.py` 改为调用 `extract_cli.py`，并将 `chart` 纳入解包结果校验目录。
3. `organizer.py` 增加 `chart` → `charts` 目录映射；manifest 与上传逻辑自动覆盖新资源。
4. 补充单测与 README 中 `charts/<歌曲ID>/{EZ,HD,IN,AT}.json` 发布结构说明。

## 期望输出

- 解包产物包含 `chart/` 目录，每首歌按难度存放 JSON 谱面。
- 整理后发布目录为 `phigros/releases/<版本>/charts/...`。
- 勾选上传时，谱面 JSON 与头像、曲绘等一并写入对象存储。

## 实际输出

- `tests/test_core.py` 通过，验证 `charts/song.id/IN.json` 进入 manifest 且 content-type 为 `application/json`。
- 完整 APK 解包验证未在本机重跑（依赖约 2.4 GiB 参考 APK 与数分钟处理时间）；逻辑与 Phigros_Resource `resource.py` chart 分支一致。
