# Phigros 字体迁移对象存储

## 改动原因

Phigros 成绩图模板原先将 12 个 TTF 随应用打包，字体原始大小合计 46,945,700 字节，显著增加 Android 与 iOS 安装包体积。现将字体改为首次使用成绩图时从固定对象存储地址下载，同时要求保持原模板字体效果、校验远端内容完整性，并避免字体未齐全时使用系统字体导出错误成图。

## 具体实现

- 新增固定版本 `v1` 字体清单，记录 12 个字体的 CSS 文件名、压缩包名、远端 URL、压缩与解压大小、两级 SHA-256、压缩包入口名及核心字体标记。
- 新增文档目录字体缓存服务，将字体持久保存到 `rranker/phigros-fonts/v1/font/`；`phi.ttf` 与 `Aldrich-Regular.ttf` 并行准备，其余字体单线程后台补齐。
- 下载过程使用临时文件，并依次验证 ZIP 大小、ZIP 哈希、唯一入口、TTF 大小与 TTF 哈希；成功后移动到正式路径，失败时清理残留。并发调用复用字体任务，损坏缓存会重新下载。
- 保留模板 CSS 的 `./font/...` 相对路径，将临时 HTML 写入同一字体版本目录，并把该目录配置为 WebView 的文件读取范围；扩展字体全部就绪后强制重建预览。
- 成绩图页新增核心字体和 `已完成 x/12` 进度、失败提示及重试入口。核心字体就绪后允许预览，12 个字体全部就绪前禁止导出。
- 删除仓库内原有 12 个 TTF，并更新第三方资源来源与分发说明。
- 新增缓存命中、同尺寸损坏缓存、网络失败、错误 ZIP 入口、大小与哈希不符、临时文件清理、并发去重、核心优先、扩展字体重试、持久目录 HTML 及界面导出门禁测试。

## 期望输出

- 普通四 ABI Android release APK 不再包含这 12 个 TTF；以原 120,540,575 字节产物和远端 ZIP 总计 21,154,843 字节估算，APK 约为 99,385,732 字节。
- 首次进入 Phigros 成绩图页时先显示核心字体准备进度，核心字体就绪后显示预览，其余字体后台顺序下载；全部完成后恢复导出。
- 远端内容、损坏缓存或网络异常时不生成字体降级图片，并可仅重试缺失或损坏字体。
- iOS 与 Android 均从持久字体版本目录读取 HTML 和字体，不再从 Expo Asset 加载这 12 个字体。

## 实际输出

- 已移除 12 个本地 TTF，共 46,945,700 字节；源码中已无对应字体的静态 `require`。
- 本任务前置联网审计已逐一确认对象存储 12/12 个 ZIP 的大小、SHA-256、唯一入口及解压字体 SHA-256与固定清单一致。完成实现后的再次联网复检因执行环境 TLS 中断及外部执行额度限制未能重复完成。
- 字体缓存与 HTML 定向测试共 10 项通过；完整 Jest UI 测试 21 个套件、97 项通过；TypeScript 类型检查通过；ESLint 为 0 错误、6 个既有警告。
- Android/iOS 生产 Expo 导出在当前沙箱中因 Metro 子进程 `spawn EPERM` 无法执行，外部执行又被额度限制拒绝，因此新 APK 实际体积、包内资源清单及 IPA 体积仍需在可运行 Metro/Gradle/EAS 的环境复核。
