# 成绩图片五列卡片与 B50 分区

## 改动原因

成绩图片预览此前只有玩家信息，没有实际成绩内容。用户要求 Best50 和自定义类型统一采用每行五张卡片，并在卡片中完整展示封面、歌曲信息、达成率、Rating、DXScore 和达成标签；Best50 还需要明确分成 B35 与 B15 两个上下区域。

## 具体实现

- HTML 画布新增五列成绩网格，卡片顶部采用难度主题色边框的歌曲封面和歌曲 ID/名称。
- 卡片下半部分展示四位小数达成率、区内排名、定数到单曲 Rating、DXScore 实际值/理论值，以及评价、FC、FS 胶囊标签。
- Best50 按 B35/B15 分区渲染，每区使用横线包围标题，并分别从 `#1` 开始编号。
- 自定义类型在自选歌曲功能接入前默认渲染当前 B50 合集，使用同一套五列卡片但不显示 B35/B15 分隔。
- 曲库成绩合并新增谱面 `notes`，理论 DXScore 按 `notes.total × 3` 计算；无法取得音符统计时显示 `—`。
- 成绩查询键和 SQLite 成绩快照版本升级，自动重新读取并保存新增的谱面音符数据。
- 增加 HTML 内容、自定义布局、曲库音符合并、旧快照迁移和预览切换回归测试。

## 期望输出

- 1080、1440、2160 三档输出均保持五列卡片，并随行数自动增加高度。
- Best50 清晰区分 B35/B15，自定义类型可复用相同卡片布局。
- 卡片理论 DXScore 来自真实曲库谱面数据，不用估算值代替。
