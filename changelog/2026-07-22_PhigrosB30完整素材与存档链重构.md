# Phigros B30 完整素材与存档链重构

## 改动原因

此前版本虽然接入了参考模板结构，但没有完整移植头像、字体和图片素材，也没有把 `GameProgress`、`GameUser` 与 Avg 数据链完整接到成图数据中，实际输出与目标示例仍有明显差距。本次按参考项目真实 B30 成图逐项重构，不再使用近似实现。

## 具体实现

- 完整复制参考项目 `resources/html/b19`、`common`、`otherimg`、`avatar` 共 172 个文件，并补充原项目许可证与头像索引；生成 109 个 PNG 头像的 Metro 静态资源表，确保 Expo 可以将中文、空格和特殊字符文件名的头像全部打包。
- 直接加载原项目 B19 CSS、12 个字体、课题模式图、评价图标、Data 图标、头像和兜底图；HTML DOM 按 `b19.art` 的 class 与层级输出玩家头部、难度统计、Phi/Best 卡片、Avg 条、OVER FLOW 和原页脚。
- 按原项目 `GameProgress` 二进制布局解析 flags、课题模式、五级 Data 单位和章节解锁字段；按 `GameUser` 读取玩家当前头像、背景与显示设置，并使用云端玩家名及存档更新时间。
- 对接原项目 Avg 接口与 Higher、Lower、Hyper、Finished 判定；推分建议采用原项目的 B27 截止与满分回退语义，接口失败时降级为不显示 Avg，不阻塞成图。
- 生成图片页继续沿用舞萌板块的类型、筛选、头像/背景、分辨率、预览、分页和导出组件布局，只替换 Phigros 数据与参考模板渲染内容。
- 修正统计表顺序为参考图的 `EZ / HD / IN / AT`，Data 文本按 `PiB / TiB / GiB / MiB / KiB` 从高到低显示。

## 期望输出

- 成图视觉与参考项目一致，原字体、头像、挑战段位、评价图标、曲绘卡片、Avg 标签、OVER FLOW 和页脚均正常显示。
- 头像下方展示存档真实 Data，例如 `386MiB 289KiB`；玩家头像和背景来自当前存档并可在舞萌式配置页中切换。
- Android 打包包含全部参考资源，缺少远程 Avg 或曲绘时仍可使用本地素材完成成图。

- 使用实际浏览器按 1080 导出视口渲染：原字体生效，33 张测试卡片、30 个 Avg、1 个 OVER FLOW、1 份头部和 1 份页脚全部存在，图片加载失败数为 0；视觉结构与用户提供的目标图一致。
