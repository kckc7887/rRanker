# 2026-07-11 产品路线图与概念 Demo

## 改动原因

项目已经具备水鱼查分、微信公众号抓取和 Windows WebUI 的概念验证，但缺少统一的产品阶段、实现优先级、技术边界和正式移动端界面方向。本次需要先明确首版范围，再用静态 HTML 验证首页、B50、查歌与工具箱的信息结构。

## 具体实现

- 新增 `ROADMAP.md`：
  - 明确 P0–P3 实现优先级。
  - 拆分 M0 工程口径、M1 只读查分、M2 同步闭环、M3 工具箱、M4 推荐验证。
  - 规划 React Native/Expo 客户端、TypeScript provider 层、SQLite 本地数据和 Python Windows 同步助手。
  - 补充建议目录、验收条件、数据许可、B50 口径、代理安全和隐私风险。
- 新增 `demo/index.html`：
  - 单文件、无框架依赖的响应式产品概念页。
  - 展示玩家概览、B35/B15 信息结构、高 RA 成绩、查歌入口和工具箱。
  - 昵称、DX Rating、记录数与四条成绩来自 `diving-fish-demo/divingfish_demo.json`。
  - 封面沿用现有 WebUI 的落雪 CDN song id 映射，并保留音符占位。
  - 明确标注 B50 尚需歌曲版本字段校验，避免把示意分组当成真实结果。
- 更新 `TODO.md`：按项目约束记录用户原始意图、目标、价值、思路、依赖风险、优先级和暂缓范围。
- 更新 `README.md`：增加 ROADMAP 与概念页入口。
