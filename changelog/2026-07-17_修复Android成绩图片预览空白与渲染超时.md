# 修复 Android 成绩图片预览空白与渲染超时

## 改动原因

部分 Android 系统搭载的 WebView 版本较旧，无法解析可选链语法，或不提供 `ResizeObserver`。成绩图片 HTML 的布局脚本会因此提前中断，表现为预览窗空白，并且导出流程收不到渲染完成消息，最终提示“图片渲染超时”。此外，任一远程装扮图片请求长时间不结束时，原有资源等待会一直阻塞到原生层 30 秒超时。

## 具体实现

- 将 WebView 内向原生发送消息的可选链调用改为兼容性检查，避免旧版 JavaScript 引擎解析失败。
- 仅在系统支持时启用 `ResizeObserver`，保留 `MutationObserver`、窗口尺寸变化和资源完成后的布局重算作为降级路径。
- 为视口宽高提供非零回退值，避免 WebView 首次布局尺寸暂不可用时把画布缩放为不可见。
- 为关键绝对定位样式补充 `top/right/bottom/left`，兼容不支持 CSS `inset` 的旧版 WebView。
- 为字体与图片等待增加 5 秒兜底，超时后按已加载素材完成布局并发送渲染完成消息；随后补发一次消息，降低部分 Android WebView 原生桥接建立较晚导致的丢信号概率。
- 关闭预览分页列表的子视图裁剪，避免 Android 对 WebView 进行错误裁剪后留下空白页面。
- 更新成绩图片 HTML 单元测试，覆盖旧 WebView 降级、资源等待兜底和非零视口缩放逻辑。

## 期望输出

在 WebView 能力较旧或个别装扮素材加载异常的 Android 系统上，成绩图片预览仍能显示，导出流程能够收到渲染完成消息并继续生成 PNG，不再等待 30 秒后报“图片渲染超时”。

## 实际输出

- 完整 Vitest 单元测试通过：40 个测试文件、173 条测试全部通过。
- 成绩图片 Jest UI 测试通过：1 个测试文件、6 条测试全部通过。
- TypeScript 类型检查通过。
- ESLint 检查通过。
- Jest 运行期间仍打印既有的 `VirtualizedList`/React `act(...)` 警告，但不影响测试通过，且本次未新增相关状态更新逻辑。
