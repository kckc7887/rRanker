# Android 详情交互、同步与封面修复

## 改动原因

Android 端出现三处回归：舞萌歌曲详情页左上角返回按钮和右上角收藏按钮无法点击；水鱼账号可以通过账密完成绑定，但用户主动同步成绩不可靠；生成成绩图片时封面缓存进度虽然完成，WebView 中仍缺少歌曲封面。iOS 端现有行为正常，需要只收口跨平台实现差异并保持 iOS 行为不变。

## 具体实现

- 将歌曲详情页的页面内标题栏叠层移动到滚动内容之后渲染，并增加 Android `elevation`，使返回与收藏按钮同时处于绘制层和命中测试层的最上方。
- 用户主动同步前取消仍在运行的总览后台刷新，避免登录后自动刷新与手动同步并发竞争。
- 手动同步直接使用 Zustand 中当前激活账号的 Import-Token 会话，不再从分账号映射二次查找；曲库尚未就绪时主动等待曲库查询，失败时给出明确错误，不再静默跳过水鱼强制回读。
- 将水鱼回读服务改为静态导入，避免 Android Hermes 运行时才加载 SQLite 回读模块的时序差异。
- 按 Expo SDK 54 契约，将 `expo-image` 在 Android 返回的 `/data/...` 绝对缓存路径规范化为 `file:///data/...`，再交给 `expo-file-system` 读取 Base64，保证封面可以作为 data URI 传入 WebView。
- 新增 Android 缓存路径、详情页返回按钮、当前 Import-Token 手动同步的回归测试，并保留收藏交互断言。

## 期望输出

- Android 舞萌歌曲详情页的返回和收藏按钮均可正常点击，iOS 行为不变。
- 水鱼账密绑定后点击“同步数据”会稳定读取当前账号最新成绩并更新本地快照与总览。
- 成绩图片缓存进度结束后，预览和导出的歌曲卡片均显示对应封面。
