# 歌曲详情滚动区按钮点击修复

## 改动原因

舞萌歌曲详情采用纵向页面滚动嵌套横向难度卡片轮播。iOS 在两层滚动手势竞争时会取消滚动区内 React Native 原生 `Pressable` 的点击，导致难度卡片中的练习清单、谱面确认搜索、标签按钮，以及卡片下方的重试、别名展开和歌曲标签按钮无法触发。

## 具体实现

- 将难度卡片中的练习清单和谱面确认搜索按钮改为与横向轮播相同手势体系的 `GesturePressable`。
- 将标签编辑器放入局部 `GestureHandlerRootView`，删除标签和添加标签按钮改用 `GesturePressable`，同时覆盖谱面标签与歌曲标签两个使用场景。
- 将收藏品重试和歌曲别名展开按钮放入各自的局部手势根节点并改用 `GesturePressable`。
- 保持页面外层纵向滚动的默认触摸取消行为，避免重新引入详情页底部无法竖向滚动的问题。
- 返回和收藏按钮位于滚动区外，继续使用 React Native 原生 `Pressable`，不扩大手势根节点范围。
- 扩充 UI 回归测试，同时验证谱面标签和歌曲标签的添加回调。

## 期望输出

歌曲详情中难度卡片及其下方的练习、搜索、标签、重试和别名展开按钮均可正常点击；难度卡片左右滑动和详情页上下滚动保持可用。

## 实际输出

滚动区内的交互按钮已统一使用局部 gesture-handler 点击处理，练习清单、搜索谱面确认和两类标签操作均能触发对应回调；纵向滚动配置未回退到会导致底部卡死的方案。
