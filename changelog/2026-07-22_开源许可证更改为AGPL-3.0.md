# 2026-07-22 开源许可证更改为 AGPL-3.0

## 改动原因

项目需要将自有代码的开源许可证从宽松的 MIT License 调整为带网络交互源码提供义务的 GNU Affero General Public License v3.0。

## 具体实现

- 使用 GNU 官方 AGPLv3 完整许可文本替换根目录 `LICENSE`。
- 将 README 的许可证说明更新为 GNU Affero General Public License v3.0，并明确 SPDX 标识为 `AGPL-3.0-only`。
- 在移动端 npm 根包及锁文件根包元数据中加入 `AGPL-3.0-only` 声明。
- 保留 Phigros B30 第三方模板与素材原有的 Apache-2.0 许可副本及第三方声明，不改变其许可归属。

## 期望输出

- GitHub 等代码托管平台能够将项目根许可证识别为 AGPL-3.0。
- README、npm 元数据与根许可证保持一致，不再把项目自有代码声明为 MIT。
- 第三方内容继续保留并遵守各自许可证。

## 实际输出

- 根目录已采用标准 GNU AGPLv3 全文。
- README 与 npm 根包元数据均已统一为 `AGPL-3.0-only`。
- 全仓检索未发现仍将 rRanker 当前许可证声明为 MIT 的有效项目文档或元数据；历史 changelog 与依赖包许可证记录保持原样。
- 第三方 Apache-2.0 许可文件及声明未被修改。
