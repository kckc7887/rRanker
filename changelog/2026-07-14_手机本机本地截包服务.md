# 手机本机本地截包服务

## 改动原因
用户要求本地服务在手机上开（VPN 投递到本机），不再依赖电脑旁路服务。

## 具体实现
- 新增 `local-capture-service`：进程内状态机；`acceptLocalCapture` → TS 换 cookie → 玩家名探针
- 新增 `local-capture-http`：`expo-http-server` 监听 `127.0.0.1:18765`（`/v1/capture|status|authorize-url|health`）
- 上传页打开时启动本机监听；UI 读进程内状态；关闭页不停服务（去微信时仍可接收）
- 依赖：`expo-http-server`、`expo-dev-client`；`app.json` 增加 package / bundleId
- Expo Go 无法听端口，需开发构建 `npx expo run:android` / `run:ios`

## 期望输出
开发构建上，App 内可监听 18765；VPN 投递后状态面板显示玩家名。

## 实际输出
本机服务代码与上传页接线已落地；真机监听需开发构建验证。
