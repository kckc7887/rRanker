# 成绩图片预览自适应

## 改动原因

用户澄清：分辨率只决定最终输出清晰度，所有分辨率都应缩放到固定预览窗内；“高度可变”是指未来成绩行增加时由内容自动撑高，而不是允许用户手动设置高度。

## 具体实现

- 移除高度输入框和高度加减按钮。
- 预览窗固定为 3:4，切换 1080、1440、2160 输出宽度时不再改变预览窗尺寸。
- HTML 画布通过实际 WebView 视口与输出画布宽高计算 `contain` 缩放比例，始终完整居中显示。
- HTML 使用 `ResizeObserver` 与 `MutationObserver` 监测内容变化，测量逻辑高度后通过 WebView 消息上报原生页面。
- 输出高度最低为宽度的 4/3；未来添加成绩行后可自动增高，原生页面只读显示测量后的输出尺寸。
- 补充跨分辨率预览尺寸不变、自动高度消息和 HTML 自适应脚本测试。

## 期望输出

切换输出分辨率时，预览画面始终完整适配同一个预览窗；未来内容变多时，输出高度自动增长并继续整体缩放显示，无需用户管理高度。

## 实际输出

预览窗与内容高度逻辑已按上述规则调整；当前只有玩家头部，因此输出保持最小 3:4，后续成绩行接入后会自动扩展。3 项 HTML 单元测试、2 项界面测试、TypeScript 类型检查、目标文件 ESLint、差异格式检查和 Android Expo 导出均已通过。
