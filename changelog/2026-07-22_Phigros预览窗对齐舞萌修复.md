# Phigros 预览窗对齐舞萌修复

## 改动原因

此前 Phigros 成绩图片预览窗错误地跟随最终导出图片的自然高度改变比例，导致生成页面的预览尺寸、页面占用空间和分页观感与舞萌板块不一致。用户要求严格参照舞萌板块的现有实现。

## 具体实现

- 按舞萌生成图片页的实现计算预览尺寸：屏幕宽度异常时回退到 `390`，预览宽度限制在 `280` 至 `720`，高度固定为宽度的 `4 / 3`。
- 保留 Phigros 图片自然高度的测量结果，仅用于尺寸信息与最终 PNG 导出，不再影响页面中的预览框高度。
- 更新 UI 回归测试，模拟 WebView 返回 `1080 × 1215` 的真实成图尺寸，并断言预览框仍保持舞萌同款 3:4 比例。

## 期望输出

Phigros 与舞萌生成图片页使用相同尺寸和比例的预览窗；Phigros 最终图片仍按自然内容高度导出，不重新引入底部多余灰色区域。

## 实际输出

- Phigros 预览窗已固定为舞萌同款 3:4 比例，真实成图高度不再改变预览容器。
- `npm run typecheck` 通过。
- `npm run lint` 通过，保留仓库已有的 6 条 warning，无新增错误。
- 完整测试通过：Vitest 67 个文件、332 项测试，Jest 21 个套件、95 项测试全部通过。
- Expo iOS 与 Android 生产资源导出均成功。
