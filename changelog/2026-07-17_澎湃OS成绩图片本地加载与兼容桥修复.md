# 澎湃OS成绩图片本地加载与兼容桥修复

## 改动原因

澎湃OS 2.0.208.0 真机仍出现成绩图片预览空白和导出渲染超时。新增诊断信息显示“页面已载入，等待渲染，WebView 版本未知”，说明原生 `onLoadEnd` 已触发，但 HTML 没有通过 `window.ReactNativeWebView.postMessage` 上报运行时或就绪消息。用户提供的排查记录进一步指出两项高风险组合：包含大量 Base64 素材的 HTML 通过 `loadDataWithBaseURL` 整体加载，以及小米 WebView 可能错误报告支持 `WEB_MESSAGE_LISTENER`、实际却没有暴露消息对象。

## 具体实现

- 在导出按钮下方增加 WebView 内核版本和渲染状态小字，从 User-Agent 提取 Chrome/WebView 版本，并显示加载、等待渲染、渲染中、就绪、响应超时、加载失败和渲染进程终止等状态。
- HTML 主动上报 `best-image-runtime`，原生侧解析运行时版本；继续复用高度与就绪消息判断真实渲染进度。
- Android 不再把完整 HTML 交给 `loadDataWithBaseURL`，而是写入应用缓存目录后通过 `file://` URI 加载；预览和导出共用同一批本地页面，离开或重建页面时删除缓存文件。
- Android WebView 仅对本应用创建的缓存 HTML 开启文件读取，不开放文件 URL 的跨域读取能力。
- 增加安装后自动执行的 `react-native-webview` 13.15.0 补丁：小米制造商设备改用 Android `addJavascriptInterface` 兼容桥，其他设备继续使用 `addWebMessageListener`。补丁固定校验依赖版本和原始代码片段，依赖升级或上游实现变化时会直接失败，避免静默漏打。
- 将 Android `versionCode` 从 2 提升到 3，防止澎湃OS安装器把本次 APK 与上一轮同版本构建混淆或复用缓存。
- 增加 HTML 运行时消息、版本解析、状态切换、Android 缓存页面创建和回收测试。

## 期望输出

澎湃OS 设备加载成绩图片时不再把数 MiB 的 HTML 直接传给 `loadDataWithBaseURL`，并能通过兼容桥收到运行时、尺寸和渲染就绪消息；预览正常显示，导出不再等待 30 秒后超时。若仍有设备差异，按钮下方状态能明确显示 WebView 版本和停止阶段。

## 实际输出

- TypeScript 类型检查和 ESLint 全量检查通过。
- Vitest 41 个测试文件、176 项测试全部通过；Jest 7 个测试文件、32 项测试全部通过。成绩图片 UI 测试仍输出项目既有的 FlatList `act(...)` 环境提示，不影响结果。
- Expo Android 预构建成功，Gradle release 构建成功；`react-native-webview` Java 类已重新编译。
- 反汇编编译产物确认存在 Xiaomi 制造商判断，并在兼容分支调用 `addJavascriptInterface`。
- 通用 APK 包名为 `com.rranker.app`，`versionCode=3`，包含 `armeabi-v7a`、`arm64-v8a`、`x86`、`x86_64` 四个 ABI，并通过 APK Signature Scheme v2 签名校验。
- 当前环境没有澎湃OS 2.0.208.0 真机，最终预览和导出结果仍需在问题设备安装本次 APK 后确认。
