# ScoreHub 绑定二维码快路径

## 改动原因

二维码登录慢路径用昵称与 Rating 反查好友码，易出现「好友列表未找到」。ScoreHub 网页做法是先好友码登录再绑定玩家二维码，之后走快路径。rRanker 已有好友码拉分，补上绑定与本地已绑定状态。

## 具体实现

- `score-hub-client`：新增 `fetchMe`、`bindCabinetByQr`（201 成功；400 已绑定幂等；409 可读错误）
- `score-hub-account-store`：SecureStore 持久化 `friendCode` / `hasCabinetBound` / 可选 `token`
- 好友码编排：`update_score` + sync 后若传入绑定二维码则 `PUT /me/cabinet`；失败不阻断写出
- 二维码编排：本地未绑定直接拒绝并引导好友码绑定；登录后用 `GET /me` 刷新绑定状态
- `UploadDataSheet`：未绑定展示绑定输入；二维码模式引导切回好友码；已绑定默认推荐二维码

## 期望输出

- 首次：好友码 + 玩家二维码 → 拉分 → 绑定 → 写出，本地记已绑定
- 之后：神秘二维码可直接上传
- 相关单元测试与 UI 测试通过
