# Phigros 成绩页数据管线与总览状态修复

## 改动原因

1. 成绩页使用独立的 `usePhigrosRecords` hook 与 overview 的 `useGameData` 缓存分离，导致 overview 的同步按钮无法刷新成绩页
2. overview 中多个 `kind === 'maimai'` 守卫挡住了 Phigros 数据的正确展示
3. 同步按钮缺少查分器来源小字说明
4. `PhigrosScoreProvider.getRecords()` 中 `achievements` 映射有误（`acc*100` 应为 `acc`）

## 具体实现

### 修改文件

1. **`src/providers/phigros-score-provider.ts`** — 修正 `getRecords()` 中 `achievements` 映射：`entry.acc` 已是百分比值，无需 `*100`

2. **`src/components/phigros/PhigrosScoreCard.tsx`** — 重写为接受 `ScoreRecord`（而非 `PhigrosScoreEntry`），统一数据类型

3. **`app/(tabs)/records/index.tsx`** — 重写 `PhigrosRecordsScreen`
   - 改用 `useGameData()` 获取成绩数据（与 overview 共享 react-query 缓存）
   - 配合 `usePhigrosCatalog()` 获取曲名映射
   - overview 的同步按钮 → `refetch()` → 成绩页同步刷新

4. **`app/(tabs)/(overview)/index.tsx`** — 修复 Phigros 总览
   - `SourceStatus`：`phigros` 与 `maimai` 同权展示数据来源
   - `DxRatingCard`：Phigros 显示 RKS 数值，不再回退到 "—"
   - 同步按钮：Phigros 显示 "TapTap 云存档" 小字说明
   - `数据状态` 卡片：Phigros 显示来源/曲库/更新时间
   - `syncProviderHint`：新增 `'phi-taptap'` → `'TapTap 云存档'`

### 删除文件

5. **`src/hooks/use-phigros-records.ts`** — 不再需要，功能由 `useGameData` + `usePhigrosCatalog` 替代

## 期望输出

- overview 的同步按钮点击后成绩页数据同步刷新
- overview 正确显示 Phigros 的 RKS、数据来源、更新时间
- 同步按钮下方显示 "TapTap 云存档" 小字
- lint/typecheck/test 全部通过
