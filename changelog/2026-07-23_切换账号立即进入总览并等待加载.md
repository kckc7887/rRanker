# 切换账号立即进入总览并等待加载

## 改动原因

切换到已有 React Query 缓存的账号时，界面会先渲染缓存内容并叠加重拉，造成卡顿；切换到无缓存账号时，可能停在其他页面或仅在总览转圈，体验不连贯。期望切换后立刻进入对应游戏的总览账号页，并在数据就绪后再展示内容。

## 具体实现

1. 新增 `switchBoundAccount`：切换前同步 `clearAccountDataQueries`（`removeQueries`），再更新会话与 activeAccountId，并 `dismissTo` / `navigate` 到 `/(tabs)/(overview)`。
2. 总览账号切换 Sheet、游戏管理「切换到此账号」统一走该助手。
3. `useSyncOnAccountSwitch` 改为清缓存；若 `switchBoundAccount` 已清过则跳过，避免打断刚发起的请求。
4. 本地/示例账号新增仍走原 `selectBoundAccount` + 命名弹层路径，不强行跳转总览。

## 期望输出

- 从总览或游戏管理切换账号后，立即进入该游戏总览账号页并显示加载态。
- 有无本地 RQ 缓存都不再边展示边重拉导致卡顿；加载完成后展示新账号数据。
