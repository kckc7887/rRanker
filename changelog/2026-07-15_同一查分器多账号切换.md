# 同一查分器多账号切换

## 改动原因

同一查分器（如水鱼）登录第二个账号时，旧实现会用单会话 SecureStore / `setSession` 整槽替换，导致第一账号被顶掉。产品需要支持同一游戏、同一查分器下保存并切换多个账号。

## 具体实现

1. **SecureStore v2 vault**（`secure-session-store.ts`）：按 `boundAccountId` 存多份 `ProviderSession`；支持 `upsertAccount` / `removeAccount` / `setActiveAccountId`；兼容旧单 key 迁移。
2. **session-store**：`sessionsByAccountId`；`setSession` 按玩家 id upsert，不再清空其他账号；`removeBoundAccount` 只删指定账号；启动 `restoreSession(() => loadVault())`。
3. **登录**：`ProviderLoginSheet` 验证后 `upsertAccount`，文案改为可并存多账号。
4. **设置页**：列出全部已绑定舞萌账号，可逐个解绑 / 切换当前账号。
5. **上传**：`resolveUploadTargets` / 上传循环按账号各自取 Import-Token。
6. **查询**：相关 React Query key 增加 `activeAccountId`，切换账号不串数据。

## 期望输出

- 同一水鱼登录两个不同玩家账号后，设置页与总览切换列表均保留两个入口。
- 切换后成绩请求使用对应 session；上传勾选多个账号时分别使用各自凭证。
- 单测覆盖多账号 set/select/remove/restore 与上传目标判定。

## 实际输出

- 实现如上；`session-store` / `session-validation` / `score-hub-upload` / `m3-settings` 相关单测与 `tsc` 通过。
- SQLite 成绩快照仍为单槽缓存：切换账号主要依赖网络 refetch（query key 已隔离），按账号分库缓存可后续再做。
