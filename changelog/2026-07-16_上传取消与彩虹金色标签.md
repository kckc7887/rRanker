# 上传取消与彩虹金色标签

## 改动原因

- 好友码成绩已经上传且账号同步成功后，仍可能因为水鱼返回数据的曲名、宴会场谱面或最终一致性差异，被严格逐条校验误判为“同步失败”。
- 上传开始后缺少中止入口，用户只能等待完整流程结束。
- 应用内与导出图片中的彩虹、金色等级标签样式不统一，需要采用用户提供的多层渐变胶囊样式，同时保留达成率彩虹文字原样。

## 具体实现

- 将水鱼账号刷新中的逐条回配作为最终一致性等待条件；只要账号数据能够成功读取，就在重试结束后保存最新快照，不再把可读且已同步的数据误报为失败。
- 根据项目内水鱼 API 文档确认 `level_index` 只支持 0 至 4，并在上传映射阶段跳过宴会场及其他水鱼不支持的谱面，避免发送无法写入的数据。
- 将上传成功后的页面刷新错误与上传/同步错误分离，页面刷新失败只提示刷新问题，不覆盖已经完成的上传结果。
- 在上传按钮下方新增同尺寸红色“取消上传”按钮；取消信号会传递到查分器请求、水鱼上传重试与账号刷新流程，并立即停止可中止的等待或网络请求。
- 新增复用的多层渐变标签组件，使用浅色渐变底、深色渐变边框和半透明灰色叠层；应用内彩虹标签、SSS/SSS+、SS/SS+ 及导出图片对应标签统一使用该样式。
- 达成率的彩虹文字继续使用原有色板，没有随标签样式调整。
- 补充上传映射、进度状态、应用内标签颜色和导出 HTML 样式测试。

## 期望输出

- 成绩上传并完成账号同步后显示成功，不再因逐条回配差异提示同步失败。
- 上传进行中可通过红色按钮取消，界面显示“正在取消…”，后台请求与重试尽快停止。
- 应用内及导出图片的彩虹 SSS/SSS+ 标签呈现彩虹底与彩虹边框，SS/SS+ 呈现金色底与金色边框，达成率彩虹字保持不变。
