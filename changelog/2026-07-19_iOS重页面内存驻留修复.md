# iOS 重页面内存驻留修复

## 改动原因

- 上一版延迟加载只避免了标签切换动画期间立即挂载页面，但页面首次访问后会永久保持挂载。
- 总览、最佳、成绩和曲库均包含查询观察者、大列表和歌曲封面；依次访问后会让多个重页面同时常驻，内存峰值与已访问页面数量相关。

## 具体实现

- 将延迟加载容器改为聚焦生命周期容器：等待当前导航交互结束后挂载内容，失焦时取消尚未完成的挂载任务，并立即卸载当前页面内容。
- 总览、最佳、成绩和曲库统一使用该机制，因此正常情况下只保留当前聚焦的重页面视图。
- 新增曲库筛选 Zustand 状态，保存关键词、难度、类型、版本、定数上下限和中日版本名称选择。
- 扩展成绩筛选 Zustand 状态，保存中日版本名称选择。
- 页面卸载后仍保留轻量筛选条件；大列表、封面实例、页面查询观察者、滚动位置和瞬时弹窗状态不再驻留。
- 增加测试，覆盖聚焦后挂载、失焦后卸载、再次聚焦重新挂载，以及两页筛选状态的保存和重置。

## 期望输出

- 多次切换底部页面时，不再逐个积累重页面视图和图片实例。
- 快速掠过页面不会启动重内容；离开已加载页面会立即释放页面级资源。
- 返回曲库与成绩页时筛选条件保持不变。

## 实际输出

- 四个重页面已改为仅聚焦时挂载，失焦会卸载内容并取消待执行任务。
- 曲库和成绩筛选已移入轻量持久状态，不受重页面卸载影响。
- `npm run typecheck` 通过。
- `npm test` 通过：51 个单元测试文件共 233 项、15 个 UI 测试文件共 67 项全部通过。
- `npm run lint` 无错误，保留一条与本次无关的既有 import 顺序警告。
- iOS 静态导出成功，共打包 1819 个模块；未启动模拟器或设备。
