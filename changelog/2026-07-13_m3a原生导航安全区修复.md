# M3A 原生导航安全区修复

## 改动原因

用户在 iPhone 17 / iOS 27 beta 3 / Expo Go 与 Android 16.0 / API 36.1 首轮复验原生导航时发现，5 个主页面的底部内容都会被原生标签栏遮挡，并要求顶部标题栏也采用系统原生组件。

核对 Expo SDK 54 文档与本地 Expo Router 6.0.24 / react-native-screens 4.16 实现后确认：SDK 54 Native Tabs 尚无 SDK 55 才提供的自动内容安全区能力，并且无法测量原生标签栏实际高度。

## 具体实现

- 新增统一的原生标签栏内容安全区计算，iOS 使用 64 pt、Android 使用 80 dp 的平台栏高基准，并叠加设备底部安全区。
- 在 5 个 Tab 共用的 `MainTabStack` 容器一次性应用底部预留，避免逐页修改 ScrollView、FlatList 或按钮布局。
- 保留 Expo Router 原生 Stack，移除自定义标题栏背景与隐藏阴影配置，使顶部标题栏采用系统默认原生外观。
- 新增 iOS、Android、Web 与异常安全区输入测试。
- 更新 M3 验收文档、开发状态和产品待办。

## 期望输出

双端所有主页面的滚动末尾、列表项和操作按钮都位于原生标签栏上方；顶部标题、返回按钮和页面转场使用系统原生 Stack 外观与行为。

- TypeScript 类型检查通过。

- iOS Hermes 静态导出通过：1600 个模块，主包 5.04 MB。

自动化和双平台打包均未发现回归；安全区实际尺寸、滚动末尾和系统标题栏视觉仍需在用户指定的双端环境复验。
