# 合并神秘二维码到好友码上传

## 改动原因

双 Tab（好友码 / 神秘二维码）割裂：绑定与会话拉分分散，且本地只存一份 JWT，无法快捷切换多好友码。

## 具体实现

1. ScoreHub 账号存储升级为 v2 多好友码 JWT（`accounts` + `activeFriendCode`），兼容迁移 v1。
2. 上传弹窗去掉双 Tab：好友码输入框右侧「历史」列出已存 JWT 好友码；切换时 `GET /me` 刷新绑定状态。
3. 有 JWT 且未绑定：显示「通过神秘二维码绑定」展开绑定区；开始上传仍走好友码 login-requests。
4. 已绑定：隐藏绑定按钮；开始上传优先会话拉分，JWT 401/403 自动回退好友码上传。

## 期望输出

单一好友码页即可完成登录、绑定、会话/好友码分流上传；历史好友码可切换。

## 实际输出

同上；相关 Jest / Vitest 用例通过。
