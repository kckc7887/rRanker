# 舞萌多本地玩家与内置账号修复

## 改动原因

- 本地玩家与测试玩家虽然能在总览切换，但没有出现在设置页账号管理中。
- 原本地查分器使用固定 `maimai:local`，如果直接复用该账号实现“添加”，多名玩家会写入同一份 SQLite 快照并互相覆盖。
- 本地玩家缺少修改名称入口。
- 测试玩家依赖在线详细曲库，请求失败时即使设备已有曲库缓存也无法生成和显示成绩。

## 具体实现

- 设置页账号管理改为展示全部舞萌账号，包括默认本地玩家、额外本地玩家、测试玩家和远程账号；本地账号显示“修改名称”，额外本地账号还可独立删除。
- 保留常驻默认账号 `maimai:local`，新增本地玩家生成唯一的 `maimai:local:*` ID。名称列表使用 SQLite KV 持久化，成绩快照继续按账号 ID 分区保存。
- 应用启动时同时恢复本地账号元数据、各账号 Rating 和上次活跃账号；清除远程会话不会删除本地玩家。
- 好友码同步在任意本地玩家页面临时仅勾选当前本地账号，并把玩家名称与成绩写入当前账号自己的快照。
- 测试玩家读取详细曲库失败时回退到 SQLite 曲库缓存，并继续动态生成 101%、SSS+、AP+、FDX+ 全谱面成绩。
- 修正默认本地玩家错误出现“解除绑定”的条件分支；默认玩家保持常驻，只有额外本地玩家可删除。
- 补充多本地玩家元数据持久化、成绩隔离、改名、删除、重启恢复、上传目标和测试曲库离线回退测试，并更新 `TODO.md` 与移动端手验文档。

## 期望输出

- 用户可重复选择“本地查分器”添加多名本地玩家，每个玩家拥有独立名称、成绩、B50、Rating 和同步目标。
- 本地玩家和测试玩家均在设置页账号管理中可见；本地玩家可改名，额外本地玩家可删除。
- 切换、同步、重启或删除其中一名本地玩家时，不影响其他本地玩家。
- 设备已有详细曲库缓存时，测试玩家在网络异常下仍能显示动态满分成绩。
