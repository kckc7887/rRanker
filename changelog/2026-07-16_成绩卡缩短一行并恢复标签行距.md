# 成绩卡缩短一行并恢复标签行距

## 改动原因

将 DXScore 合并到“难度定数 → Rating”同一行的目的，是删除原本由 DXScore 单独占用的一行并让歌曲卡片同步变矮。此前为了保持卡片高度增加的最小高度抵消了这一效果；随后把标签行间距完全归零，又导致 Rating/DXScore 行与状态标签行贴得过紧。

## 具体实现

- 删除歌曲卡片的人工最小高度，让卡片高度重新由实际内容自然决定。
- 保持标签行 `margin-top: 0`，避免 Flex 自动外边距再次制造整行空白。
- 恢复标签行原有的响应式小上内边距，在 1080 宽导出中为 4 像素，使两行之间有正常视觉行距。
- 更新导出 HTML 回归测试，同时约束“不保留固定最小高度”“不使用自动上外边距”“保留正常小行距”。

## 期望输出

歌曲卡片比原布局减少 DXScore 独占的一行高度；Rating/DXScore 行与状态标签行之间只保留正常的小行距，不贴紧，也不再出现整行空白。

## 实际输出

导出模板已删除卡片最小高度并恢复标签行的小上内边距。歌曲卡片可随 DXScore 独占行的删除自然缩短，标签行与上方信息行之间保持固定的小间距。TypeScript 类型检查与 ESLint 均通过；Vitest 40 个测试文件共 171 项全部通过；Jest 7 个 UI 测试套件共 29 项全部通过，控制台仅保留仓库既有的 React `act(...)` 提示。
