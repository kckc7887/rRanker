# 个人曲库按游戏隔离

## 改动原因

不同游戏（舞萌 / Phigros）共用同一套个人曲库 key（如 `song:1`、`chart:1:SD:0`），收藏、标签、练习清单会跨游戏混用；Phigros 总览页「我的曲库」仍显示「暂未开放」。

## 具体实现

1. **Domain**：`UserLibraryItem` / `LibraryTarget` 增加 `gameId`；key 改为 `song:{gameId}:{songId}`、`chart:{gameId}:{...}`；备份格式升至 v3；v1/v2 导入默认归入 `maimai`。
2. **SQLite**：schema v3 增加 `game_id` 列；v2 数据迁移时旧 key 统一补 `maimai` 前缀并重写。
3. **Service / Hook**：读写按 `activeGameId` 隔离；`useUserLibrary` 暴露 `songKey` / `chartKey` 辅助方法；备份仍导出全游戏数据。
4. **UI**：总览页 Phigros 显示收藏/练习统计；曲库页只加载当前游戏曲库作标题/曲绘解析。

## 期望输出

- 舞萌与 Phigros 收藏/标签/练习互不影响。
- Phigros 总览「我的曲库」显示正确统计并可进入。
- 旧版 v2 备份导入后数据归入舞萌。

## 实际输出

- 单元测试 292 项通过（含 `user-library`、`user-library-service`、`sqlite-user-library-repository`）。
- 待设备端验证：切换游戏后总览统计与曲库列表是否即时更新。
