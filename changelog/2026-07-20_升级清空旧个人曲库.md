# 升级清空旧个人曲库

## 改动原因

按游戏隔离个人曲库后，不必把隔离前的收藏/标签/练习迁移到 `maimai`；用户要求直接清空旧数据。

## 具体实现

- SQLite 个人曲库 schema 升至 **v4**。
- 从 v1–v3 升级时：确保 `game_id` 列存在后，删除 `user_library_items` / `user_library_item_tags` / `user_library_tags`，保留标签预设，再写入 schema 版本。
- 不再重写旧 key 为 `song:maimai:…`。

## 期望输出

升级到含本改动的版本后，本地旧收藏、练习清单与条目标签为空；之后新建的数据按当前游戏隔离。

## 实际输出

- `sqlite-user-library-repository` 相关单元测试通过。
- 已处于 v3（曾迁移过旧数据）的设备也会在升到 v4 时清空一次。
