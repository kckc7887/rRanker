# iOS 标签切换与游戏管理回归修正

## 改动原因

- Expo SDK 54 原生标签导航会在导航器挂载时急切渲染全部标签页面；总览、最佳、成绩和曲库均包含查询与较大列表，快速点击多个标签时会让页面挂载和导航动画竞争主线程。
- 游戏管理原本位于设置标签的内部 Stack，进入后底部导航仍会保留。
- 上一版错误地重做了游戏管理的视觉与添加入口，超出了“仅按游戏折叠已有查分器/账号”的范围。

## 具体实现

- 新增原生标签页延迟挂载容器：页面首次聚焦后等待导航交互结束再挂载重内容；若用户在完成前快速切走，则取消该次挂载；成功挂载后保持页面状态。
- 总览、最佳、成绩和曲库接入延迟挂载，设置页保持轻量直接渲染。
- 将游戏管理从 `(tabs)/settings` 移到根级 `game-management` Stack 页面，设置入口改为跳转根级路由，因此页面不再显示底部导航。
- 恢复原账号卡片的尺寸、圆角、间距和文案样式，恢复右下角 56 像素悬浮加号、游戏选择弹窗、查分器登录及返回弹窗流程。
- 仅对已有账号按游戏增加展开/收起标题；默认展开，未绑定账号的游戏只在原添加弹窗中出现。
- 增加测试，覆盖快速切走时取消重页面挂载、根级游戏管理路由、原账号卡片与悬浮按钮样式、按游戏折叠和原添加流程。

## 期望输出

- iOS 快速切换多个底部页面时，不再因多个重页面同时首次挂载而产生严重卡顿。
- 游戏管理页面不显示底部导航栏。
- 游戏管理保持改造前的账号卡片、右下角添加按钮和添加弹窗，仅新增按已有账号所属游戏展开列表。

## 实际输出

- 四个重标签页已改为首次稳定聚焦后挂载，短暂快速切换会取消尚未开始的重渲染。
- 游戏管理已成为根级 Stack 页面，底部 NativeTabs 不再属于该页面。
- 原账号卡片、悬浮添加按钮和 GamePickerSheet 流程已恢复，并通过样式与交互测试锁定。
- `npm run typecheck` 通过。
- `npm test` 通过：50 个单元测试文件共 230 项、15 个 UI 测试文件共 67 项全部通过。
- `npm run lint` 无错误，保留一条与本次无关的既有 import 顺序警告。
- iOS 静态导出成功，共打包 1818 个模块；未启动模拟器或设备。
