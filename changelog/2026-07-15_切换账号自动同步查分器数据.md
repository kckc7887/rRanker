# 切换账号自动同步查分器数据

## 改动原因

切换游戏 / 查分器 / 同一查分器下另一账号时，仅更新内存登录态不够；若命中 React Query 缓存（`staleTime` 内），界面可能继续展示旧账号数据，且设置页切换此前也不会触发拉取。需要在账号变更后自动对应当前 provider 同步一次。

## 具体实现

1. 抽出 `invalidateAccountDataQueries`：失效 `game-data` / `score-snapshot` / `detailed-catalog` / `plates` / `songs`。
2. 新增 `useSyncOnAccountSwitch`：监听 `activeAccountId`，restore 就绪后的后续变更触发上述失效；首次基线不触发，避免启动双拉。
3. 在根布局 `QueryClientProvider` 内挂载同步钩子，总览与设置页切换共用。
4. 总览手动「同步数据」改为复用同一失效助手；切换后 `isFetching` 时按钮显示「同步中…」。

## 期望输出

- 总览或设置中切换到另一已绑定账号后，自动向该查分器拉一次成绩/曲库相关数据。
- App 冷启动 restore 不会因此额外多打一轮失效同步。

## 实际输出

- 实现如上；`invalidate-account-data` 单测覆盖 query key；与总览接线已接好。
