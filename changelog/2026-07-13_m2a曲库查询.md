# M2A 曲库查询

## 改动原因

M1 的轻量 LXNS 曲库只能支撑 B35/B15 分类，缺少别名、谱师、物量、BPM 等查歌和详情所需信息。M2A 需要在不改变水鱼成绩职责的前提下补齐可组合查询能力。

## 具体实现

- 扩展歌曲和谱面模型，加入 BPM、分类、版权、锁定/禁用、数值版本、别名、谱师及六类物量字段。
- LXNS provider 新增详细曲库与别名读取，详细曲库、别名按资源键独立写入 SQLite；损坏时只淘汰对应资源。
- 建立 NFKC + 小写的内存搜索索引，支持曲名、ID、别名、曲师、谱师及 SD/DX、难度、定数、歌曲版本、谱面版本组合筛选。
- 查歌使用短防抖、可折叠筛选区和 FlatList 虚拟化；曲绘使用 LXNS 资源地址、`expo-image` 磁盘缓存和失败占位，不做批量预取。
- 新增歌曲详情页，展示元数据、别名、各谱面定数/谱师/物量，并匹配本地水鱼最佳成绩。

## 期望输出

1300+ 曲库能够稳定滚动；所有查询条件可组合；长标题、无别名、无成绩和封面失败均有明确展示；详细曲库和别名任一失败时不破坏其他缓存。

## 实际输出

- 自动化覆盖详细 API schema、别名/谱师搜索、NFKC、组合筛选、资源缓存与歌曲详情状态。
- 最终完整测试中 Vitest 18 个文件、61 项通过；Jest Expo 的 M2 查询交互与详情测试通过。
- Android Expo 静态导出成功，生成 Hermes `.hbc` 包。
- iPhone Expo Go 与 Android Emulator 的真实网络、长列表和封面失败仍需按 `docs/mobile-testing.md` 手工复验；素材许可门禁仍保留。
