# 成绩图片自定义筛选、分页与 PNG 导出

## 改动原因

原有成绩图片只能完整生成 Best50，“自定义”仍复用 B50 数据，无法按版本、数量、达成率或 FC/FS 成就筛选；长结果也没有分页机制。头像、姓名框、称号和背景选择不会跨重启保留，预览完成后亦不能按所选分辨率保存图片。彩虹评价、彩虹称号和选择器预览还存在黄色色段、文字颜色及垂直居中不一致的问题。

## 具体实现

- 抽出共享标签主题，统一原生成绩标签、歌曲详情称号、称号选择器与 HTML 图片的彩虹渐变、实底状态色和文字色；彩虹删除黄色，统一使用灰色文字。评价、寸、FC、FS、DX/SD 和称号均使用固定高度、双向居中及对称横向留白。
- 新增 `CustomBestImageFilters` 及纯函数筛选链路，直接读取全部 `maimai.records`，支持数量（0 不限）、当前/过往版本、双版本合并或区分、0–101 最小达成率、FC/FS 互斥成就筛选和严格匹配。
- 自定义标题统一按 `{版本}{成就或Best}{实际数量}` 生成；合并模式统一排序后截取一次，区分模式对两个分区分别排序和截取，空结果仍输出 `Best0`、`AP0` 等标题。
- 新增 `BestImagePage` 分页：五列布局每页最多 50 行，分区可使用同页剩余行；跨页保留分区总标题与连续排名，每页重复玩家头部和 `第 x / y 页`。
- 预览改为横向分页 `FlatList`，限制为当前页与相邻页渲染；歌曲封面仍先由原生层串行缓存完成，再一次性注入所有页的 HTML。
- 使用 `expo-sqlite/kv-store` 按账号持久化头像、姓名框、称号和背景的当前/关闭/指定/随机模式；损坏、类型不匹配或收藏品不存在时回退当前样式。随机模式重新进入账号时重抽，并在当次预览和导出中保持固定。
- 接入 Expo 54 兼容的 `react-native-view-shot` 与 `expo-media-library`。独立导出 WebView 等待字体、封面、背景和动态高度稳定后，按 1080/1440/2160 实际像素逐页捕获 PNG，并保存到相册；加入权限、进度、错误提示、文件命名和临时文件清理。
- 更新 Expo 相册权限配置、`TODO.md`、单元测试和 UI 回归测试。

## 期望输出

- 自定义成绩能按用户条件生成正确的 BestN 或成就 N；单版本、双版本合并和双版本区分的标题、数量与排名均符合约定。
- 250 张成绩保持单页，251 张自动分为两页；预览可横向切换，导出获得全部分页 PNG。
- 账号间样式配置互不影响，随机样式在单次预览/导出中稳定，失效配置不会阻塞页面。
- 彩虹标签不含黄色、使用灰色文字，所有标签文字视觉居中。
- 导出 PNG 的宽高使用 HTML 稳定后的实际尺寸，不截取屏幕预览窗，也不会因设备像素倍率再次缩小。

## 实际输出

- Vitest：39 个测试文件、159 项测试全部通过；覆盖筛选、标题、严格/至少匹配、251 张分页、排名偏移、持久化隔离、损坏/存储故障回退、导出权限/命名/清理及 HTML 渲染完成消息。
- Jest UI：7 个测试文件、29 项测试全部通过；覆盖预览类型、分辨率、收藏品选择、称号筛选、非法输入保留预览及 1080×1440 导出捕获参数。测试渲染器仍会输出 FlatList 异步 `act` 提示，但不影响用例结果。
- TypeScript 检查通过；ESLint 无新增错误，仅保留仓库原有的总览页 Hook 依赖警告。
- Expo 54 Android Hermes 静态导出成功，共打包 1795 个模块并包含新增原生模块配置。
- 本地浏览器因安全策略拒绝打开 `file://` 视觉验收页，因此未绕过策略；1080/2160 的真实长图视觉、相册权限和高行数内存占用仍需在 iPhone Expo Go 与 Android 环境人工复验。
