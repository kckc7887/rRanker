# 封面串行缓存与样式选择增强

## 改动原因

成绩图片一次包含最多 50 张歌曲封面，直接交给 WebView 会同时触发大量 CDN 请求，出现加载缓慢或部分封面不显示。样式选择还缺少随机和关闭能力，称号列表无法按等级筛选；玩家名行高和内边距也会造成文字没有垂直居中甚至被裁切。

## 具体实现

- 新增歌曲封面预加载器：先读取 Expo Image 磁盘缓存，未命中时严格一次请求一张，缓存后通过 Expo FileSystem 转换为 data URI。
- 对歌曲 ID 去重并显示逐张缓存进度；全部封面处理完成后才一次性创建 WebView HTML。
- 下载失败的封面明确写入失败结果并显示音符占位，阻止 WebView 再次直接请求远程封面。
- 切换分辨率、背景或其他样式时复用内存与磁盘缓存，不重复下载相同封面；仅当歌曲 ID 列表内容变化时重新执行任务。
- 头像、姓名框、称号和背景选择器新增“随机”“关闭”，同时保留“玩家当前”和指定收藏品。
- 关闭头像、姓名框、称号时真正隐藏对应 HTML 元素；关闭背景时使用默认底色。随机选项记录本次抽到的具体收藏品，预览可稳定复现。
- 称号选择器增加全部、普通、铜、银、金、彩虹等级筛选，等级来自落雪收藏品 `color` 字段。
- 玩家名改为 `inline-flex`，使用固定最小高度、垂直居中和更宽松行高，避免文字出框。
- 增加封面缓存、失败占位、随机/关闭、称号等级筛选和名字框样式回归测试。

## 期望输出

- 首次进入预览时能看到封面缓存进度，完成后所有成功封面一次性出现；后续进入或切换样式直接复用缓存。
- 不再发生 WebView 同时请求全部封面的情况，单张失败也不会拖累或反复请求整个列表。
- 四类样式都可选玩家当前、随机、关闭或指定收藏品，称号能按等级快速筛选。
- 玩家名在背景框内上下居中，不裁切字形。
