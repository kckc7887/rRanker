# 舞萌成绩成就筛选与 SYNC 标签

## 改动原因

成绩页缺少 AP+/AP/FC+/FC/FDX+/FDX/FS+/FS/SYNC 成就筛选；最佳页与成绩页的 SYNC 标签样式与 FS/FS+ 不一致；成绩图片自定义模式选 SYNC 时结果恒为空。

## 具体实现

1. **共用成就状态筛选**（`domain/maimai-filters.ts`）
   - 抽出 `MaimaiAchievementStatus`、FC/FS 选项表、`normalizeMaimaiFs/Fc`、`matchesAchievementStatus`。
   - FS/FC 等级改为 1-based，避免 SYNC/FC 最低档位用 `0` 时被假值判断误伤；匹配时用 `=== null` 而非 `!actual`。
   - 支持 `rawFs`/`rawFc` 回退，并规范化 `fdx`/`fdxp` 与大小写空白。

2. **成绩页筛选**
   - `records-filter` 增加 `achievementStatus`。
   - `MaimaiFilterBar` 在成绩页展示成就两行芯片；收起摘要带上成就标签。

3. **SYNC 标签样式**
   - `ScoreVisuals`：`fs === sync` 与 FS/FS+ 同为蓝色流动玻璃标签；取值先 `trim` 再匹配。
   - 脱敏夹具部分成绩写入 `sync`，便于最佳/成绩列表可见。

4. **成绩图片**
   - `best-image-custom` 改用共用匹配函数；补 SYNC 筛选单测。

## 期望输出

- 成绩页可选成就筛选，SYNC 表示「至少 SYNC」（含 FS/FDX 等更高档）。
- 最佳页、成绩页在 `fs=sync` 时显示与 FS/FS+ 同风格的 SYNC 标签。
- 成绩图片选 SYNC 能筛出含 sync/fs/fsp/fsd/fsdp 的成绩，严格模式仅精确 SYNC。

## 实际输出

相关 vitest / jest（maimai-filters、best-image-custom、records-filter、m4-score-lists、m2-query）已通过。
