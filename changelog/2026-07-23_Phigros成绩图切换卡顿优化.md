# 优化 Phigros 成绩图 Best30/自定义切换卡顿

## 改动原因

切换 Best30 与自定义时会清空曲绘、重跑字体准备并连带重载头像背景，预览整页空白等待，体感卡顿很久。

## 具体实现

1. 新增 `partitionPhigrosIllustrationCache`：会话内曲绘增量复用，仅下载缺失 songId；切换时先用缓存立刻出预览。
2. 字体准备 effect 只依赖 `neededFontKey`（不再因 `neededFontEntries` 新引用重复执行）。
3. 头像/背景与曲绘拆分加载，按 `styleAssetKey` 去重，避免仅切类型时重拉。
4. Acc 均值与 WebView sources 不再在切换时无谓清空到空白态；类型切换使用 `startTransition`。

## 期望输出

- Best30 ↔ 自定义切换时分段控件立即选中，重叠曲目不重新拉曲绘。
- 字体集不变时不重新准备字体、不禁用导出门禁。

## 实际输出

- Vitest：`partitionPhigrosIllustrationCache` 用例通过。
- Jest：切换自定义后 `loadPhigrosIllustrations` 不再被调用（缓存命中）。
