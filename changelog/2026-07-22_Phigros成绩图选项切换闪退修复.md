# Phigros 成绩图选项切换闪退修复

## 改动原因

Phigros 参考模板的 12 个字体原始总量约为 44.77 MiB。此前为解决双平台素材加载问题，将字体和 PNG 一并转换为 Base64 Data URI，导致每一页预览 HTML 仅字体部分就膨胀到约 59.69 MiB。切换类型、头像、背景、OVER FLOW 数量或分辨率时会重建 HTML 与 WebView，旧、新内容短暂并存后触发 iOS 与 Android 内存崩溃。

## 具体实现

- 保留头像、课题模式、评价、Data 图标和兜底背景的 PNG Data URI，继续保证这些图片在两个平台稳定显示。
- 将原模板字体改为通过 Expo Asset 下载并解析得到的本地缓存文件 URI，不再把字体内容复制进每份 HTML。
- 为 iOS 内联 HTML 增加可配置的 `baseUrl`，Phigros 预览和导出使用 Expo 缓存目录作为基准目录，使 WKWebView 可以加载同目录中的字体文件。
- Android 继续使用缓存目录中的临时 HTML，并复用现有文件访问权限读取字体。
- 增加回归断言，确认字体保留文件 URI、图片保留 Data URI、iOS HTML 使用缓存目录 `baseUrl`，同时保持统一成绩图来源工具的默认行为兼容。

## 期望输出

- 在 Phigros 生成图片页选择任意配置时不再闪退。
- 原参考项目字体正常显示，头像、评价图片、课题模式图与玩家 Data 继续正常显示。
- iOS 与 Android 均能打包字体资源并使用现有预览、分页和 PNG 导出流程。

## 实际输出

- 12 个字体不再 Base64 内嵌，单份 HTML 减少约 59.69 MiB 的重复字体文本。
- `npm run typecheck` 通过。
- `npm run lint` 通过，0 个错误；保留基线已有的 6 个警告。
- `npm test` 通过：67 个 Vitest 文件共 332 条测试、21 个 Jest 文件共 95 条测试全部通过。
- Expo iOS 与 Android 生产资源导出均通过，12 个原字体均作为独立资源进入对应平台产物。
