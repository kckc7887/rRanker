# M3A 原生底部导航

## 改动原因

现有 5 项底部导航由 Expo Router JavaScript Tabs 渲染。用户明确要求将底部导航也替换为 Android/iOS 各自的原生系统组件，同时保持既有入口、顺序和二级页面导航体验。

## 具体实现

- 使用 Expo SDK 54 与 Expo Router 6.0.24 提供的 `expo-router/unstable-native-tabs` 替换 JavaScript Tabs。
- 保留“总览、B50、成绩、查歌、设置”5 个静态入口及原顺序。
- iOS 使用 SF Symbols，并为支持的入口配置默认/选中符号；Android 使用同一套 Ionicons 轮廓图标配合原生选中着色，规避 SDK 54 不支持 Android 双态图标的问题。
- 将每个 Tab 页面放入独立原生 Stack，恢复 JavaScript Tabs 迁移后不会自动提供的系统标题栏。
- Android 返回行为按访问历史切换 Tab；未为“查歌”启用会改变 iOS 排列方式的特殊 search role。
- 更新总览页面 UI 测试引用路径，并同步更新产品待办状态。

## 期望输出

Android 与 iOS 均使用平台原生底部导航组件；五个入口、页面标题和根级二级页面转场保持稳定，不再依赖 JavaScript 模拟底栏或模拟标题栏。
