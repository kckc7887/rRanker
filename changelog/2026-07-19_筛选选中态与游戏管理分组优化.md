# 筛选选中态与游戏管理分组优化

## 改动原因

- 原筛选器中“全部”、难度和谱面类型的选中态使用了不同视觉方式，辨识一致性不足。
- 游戏管理页先平铺账号，再通过右下角按钮打开游戏和查分器选择器，账号所属游戏与查分器的层级不够直观。

## 具体实现

- 新增共享筛选标签外框：始终预留两像素透明边框与内间距，选中时切换为蓝色圆角外框；外框比标签略大，并分别贴合胶囊难度标签与圆角矩形类型标签，因此切换状态不会引起尺寸重排。
- “全部”、五档难度和 SD/DX 类型统一复用该外框，同时保留各标签原有底色、文字与渐变样式。
- 游戏管理页改为按游戏显示折叠卡片，默认展开舞萌 DX；游戏摘要显示查分器与账号数量。
- 展开游戏后按查分器分组显示绑定说明、添加或绑定入口，以及该查分器下的账号卡片。
- 测试游戏等无需查分器的游戏直接展示账号；未开放游戏显示待开放状态。
- 保留远程查分器登录、本地玩家新增与改名、测试账号选择、账号切换、删除和解绑流程，并移除页面内重复的游戏选择弹窗与悬浮添加按钮。
- 更新 UI 测试，覆盖统一选中外框、游戏折叠、查分器分组、绑定入口和既有账号操作。

## 期望输出

- 所有胶囊式筛选标签被选中时都有尺寸一致、贴合形状且略大的外框，切换时不发生布局跳动。
- 设置 → 游戏管理直接按游戏展开查分器和账号列表，用户可在对应分组内完成账号管理。
- 现有登录、切换、重命名、删除与解绑功能保持可用。

## 实际输出

- 筛选器的“全部”、难度和类型标签已统一使用共享选中外框。
- 游戏管理已改为“游戏 → 查分器 → 账号”内联折叠结构，并保留全部既有账号操作。
- `npm run typecheck` 通过。
- `npm test` 通过：50 个单元测试文件共 230 项、13 个 UI 测试文件共 65 项全部通过。
- `npm run lint` 通过且无错误；保留一条与本次改动无关的既有测试文件 import 顺序警告。
- 按要求未启动模拟器或设备。
