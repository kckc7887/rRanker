# M4 单曲详情交互与本机清理修正

## 改动原因

- “同时删除个人数据”会并发写入同一个 SQLite 数据库，个人数据独占事务可能让快照清理报 `database is locked`，从而出现只删除个人数据、内存登录态与缓存仍保留的半完成状态。
- LXNS 对歌曲 `#363 / Oshama Scramble!` 返回细分版本号 `15007`，现有代码只做精确版本键查找，因此详情页直接显示数字且国服/日服切换没有有效名称。
- 透明原生标题栏覆盖了封面内的绝对定位收藏按钮，导致右上角收藏无法点击。
- 同一首歌的 SD/DX 谱面被平铺进一个难度轮播，不符合整组切换的交互；谱面元数据、物量和别名布局也需要按本轮视觉要求收口。

## 具体实现

- 本机清理改为按“凭据 → 快照缓存 → 可选个人数据”顺序执行，避免两个 SQLite 仓储并发写入；每项独立记录失败，无论单项是否失败都退出内存登录态并移除 `score-snapshot`、`songs`、`detailed-catalog`、`plates` 查询缓存，保留个人曲库查询边界。
- 按 LXNS 官方前端规则，将歌曲和谱面的细分版本号向下匹配到“不大于原值的最大主版本”；`15007` 归一为 `15000 / ORANGE PLUS`，日服名称为 `maimai ORANGE PLUS`，同时修正版本筛选与统计使用的 ID。
- 将歌曲收藏按钮移入 `Stack.Screen` 的原生 `headerRight`，使用固定圆形点击区域、爱心状态和无障碍标签。
- 单曲同时存在 SD/DX 时默认只展示 DX 全难度；点击难度标签下方的 SD/DX 小标签后整组切换并重新定位 MASTER。SD 使用蓝底白字，DX 使用白底与橙黄渐变字，双类型时显示灰色 `·点击切换·`。
- 卡片底部只保留谱师；物量改为 `TAP / HOLD / SLIDE / TOUCH / BREAK / 总计` 两行六列表格，不再展示谱面版本。
- 歌曲别名默认限制一行，通过真实文本布局测量只在溢出时显示展开/收起按钮。
- 将“从具体 SD 谱面入口进入时默认展示 SD”按用户要求记录到 `TODO.md`，本轮未提前实现。

## 期望输出

- 选择“同时删除个人数据”会确实尝试清除凭据、缓存和个人数据，不再因 SQLite 并发导致只删个人数据；若单项失败，页面明确指出失败部分，其余项目仍完成清理并退出登录态。
- 歌曲 `#363` 默认显示 `ORANGE PLUS`，点击版本切换后显示 `maimai ORANGE PLUS`，不再出现裸数字 `15007`。
- 右上角收藏可正常点击；SD/DX 不再混排，默认 DX，标签可整组切换。
- 谱师、物量表格与一行别名布局符合本轮要求，已有达成率、FC/FS、Rating、练习清单和标签功能保持可用。
