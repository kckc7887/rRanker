# Phigros 成绩图原字体读取修复

## 改动原因

上一版为避免选项切换时重复构建约 60 MiB 的 Base64 字体，将原字体改为 Expo 缓存文件 URI；但 iOS 的内联 HTML 走 `loadHTMLString` 分支，该分支不会使用 WebView 的 `allowingReadAccessToURL`，仅设置缓存目录 `baseUrl` 仍无法稳定读取字体，导致生成图再次回退为系统字体。

## 具体实现

- Phigros 预览与导出在 iOS、Android 上统一将 HTML 写入 Expo 缓存目录，并通过文件 URI 加载。
- 继续将全部 12 个参考项目原字体作为独立 Expo Asset 缓存文件，通过模板中的绝对文件 URI 引用，不恢复字体 Base64 内嵌。
- 继续以内嵌 Data URI 加载头像、评价图、课题模式图、Data 图标和兜底背景，避免图片资源回归。
- 保留舞萌页面原有 Android 文件来源函数名作为兼容别名，不改变舞萌的预览与导出行为。
- 更新来源单测与 Phigros 页面回归测试，覆盖跨平台缓存 HTML、文件清理、原字体 URI、图片 Data URI 和缓存目录读取权限。

## 期望输出

- iOS 与 Android 的 Phigros 预览、导出均使用参考项目原字体。
- 任意配置切换继续保持稳定，不重新引入大体积 Base64 HTML 导致的闪退。
- 头像、评价图、课题模式图和玩家 Data 保持正常显示。

## 实际输出

- Phigros 双端均改为缓存 HTML 文件来源，WebView 可进入带缓存目录读取权限的本地文件加载路径。
- `npm run typecheck` 通过。
- `npm run lint` 通过，0 个错误；保留基线已有的 6 个警告。
- `npm test` 通过：67 个 Vitest 文件共 332 条测试、21 个 Jest 文件共 95 条测试全部通过。
- Expo iOS 与 Android 生产资源导出均通过，全部 12 个参考字体继续作为独立资源进入平台产物。
