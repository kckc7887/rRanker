# 成绩图片标签行空白间隔修复

## 改动原因

上一轮将 DXScore 数值合并到“难度定数 → Rating”行后，为保持歌曲卡片尺寸增加了最小高度，但卡片底部标签行仍使用 `margin-top: auto`。Flex 布局因此把卡片剩余高度全部放在 Rating/DXScore 行与标签行之间，视觉上保留了一整行无内容的空白。

## 具体实现

- 保留歌曲卡片最小高度，继续满足卡片尺寸不变的要求。
- 移除标签行用于吸收剩余空间的自动上外边距，并将其上内边距归零。
- 卡片剩余高度改为自然留在最后一个内容行之后，不再插入 Rating/DXScore 行与标签行之间。
- 更新导出 HTML 回归断言，明确禁止标签行恢复 `margin-top: auto`。

## 期望输出

“难度定数 → Rating / DXScore 实际值 / 理论值”行与下方状态标签行直接相邻，中间不再出现空白行，同时歌曲卡片和标签尺寸保持不变。

## 实际输出

导出模板已移除标签行的自动上外边距和上内边距；自动化测试确认标签行紧邻 Rating/DXScore 行，歌曲卡片最小高度与所有标签尺寸未修改。TypeScript 类型检查与 ESLint 均通过；Vitest 40 个测试文件共 171 项全部通过；Jest 7 个 UI 测试套件共 29 项全部通过，控制台仅保留仓库既有的 React `act(...)` 提示。
