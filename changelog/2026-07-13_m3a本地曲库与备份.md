# M3A 本地曲库与备份

## 改动原因

M2 已完成，用户明确要求开始 M3A。现有工程只有远端成绩、曲库和资源快照，没有独立的个人收藏、练习清单、标签和跨端备份能力；清除缓存也无法让用户选择是否保留个人整理结果。

## 具体实现

- 新增歌曲收藏、谱面练习清单和歌曲/谱面标签领域模型；使用规范化歌曲 ID、谱面类型和难度序号生成稳定键。
- 标签执行 NFKC、空白折叠和大小写无关去重，限制 24 个字符、每项目 30 个标签；空状态且无标签的条目自动删除。
- 新增独立 SQLite 用户条目、标签、关联表与 schema version；更新、恢复和清空使用独占事务，不复用远端缓存表。
- 保持 5 个主 tab；总览新增“我的曲库”入口，查歌支持收藏，歌曲详情支持练习清单和标签，“我的曲库”支持类型与标签过滤。
- 新增严格的 `rranker-user-data` v1 JSON 备份；只包含个人数据白名单，默认合并并允许显式替换。
- 直接安装 Expo SDK 54 兼容的 `expo-document-picker@14.0.8`、`expo-file-system@19.0.23`、`expo-sharing@14.0.8`；导入限制 1 MiB，文件选择复制到缓存，使用后清理临时文件。
- 设置页清除操作每次询问“仅凭据与缓存”或“同时删除个人数据”，并提供备份导入/导出入口。
- 补充领域、service、SQLite、文件服务和 React Native 页面测试，并更新 M3 双端手验清单。

## 期望输出

- 收藏、练习清单和标签不随登录切换或远端缓存清理丢失。
- 曲库暂不可用时仍保留个人条目并显示稳定 ID。
- 备份不包含 token、cookie、玩家账号、成绩和曲库缓存；损坏或未知版本文件不会修改原数据。
- 默认合并保留本机标签写法并合并状态/标签，替换恢复在事务中完整提交或回滚。
- Android/iOS 共用同一领域、存储和页面实现。
