# iOS 标签页状态缓存与列表限窗

## 改动原因

- 上一版在标签失焦时卸载整个重页面，虽然阻止了多张完整页面同时驻留，但每次回切都必须从空占位重新创建视图，因此会出现明显白闪。
- 用户所说的“快照”是页面状态缓存，不是图片截图；需要保留当时的页面实例、滚动位置和有限的可视卡片窗口。

## 具体实现

- 将仅聚焦挂载容器替换为标签页状态缓存容器：重页面首次聚焦并等待导航交互结束后只激活一次，之后失焦不卸载，回切直接使用同一个页面实例。
- 总览、最佳、成绩和曲库统一接入状态缓存容器；不增加截图、临时图片或磁盘图片缓存。
- 新增统一虚拟列表缓存参数，曲库、成绩和 B50 使用首屏 8 项、每批 4 项、50 毫秒批次间隔和 3 个视口窗口。
- Android 保留离屏原生视图裁剪；iOS 不启用该裁剪，因为 React Native 0.81 文档说明它不显著释放内存且可能导致 iOS 内容缺失，iOS 的内存边界改由较小的虚拟窗口控制。
- 增加测试，验证页面内部状态在失焦和再次聚焦后保持、页面只执行一次延迟激活，以及三张列表采用一致的有限缓存窗口。

## 期望输出

- 已访问的重页面之间切换时不再闪白，筛选、输入、滚动位置和组件状态保持不变。
- 多次切换不会重复创建页面实例，列表只保留可视区及附近少量卡片，常驻组件数量保持有界。
- 不生成或保留任何页面截图。

## 实际输出

- 四个重标签页已改为首次访问后保留同一页面实例，失焦后不卸载，回切无需重新挂载。
- 曲库、成绩和 B50 已统一收紧为 3 个视口的虚拟窗口，并把滚动补充批次限制为 4 项。
- `npm run typecheck` 通过。
- `npm test` 通过：51 个单元测试文件共 233 项、15 个 UI 测试文件共 67 项全部通过。
- `npm run lint` 无错误，保留一条与本次无关的既有 import 顺序警告。
- iOS 静态导出成功，共打包 1820 个模块；未启动模拟器或设备。
